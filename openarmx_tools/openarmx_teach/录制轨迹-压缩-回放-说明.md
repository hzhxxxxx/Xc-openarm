## 快速开始

### 完整工作流程

```bash
# 1. 启动仿真环境
cd /home/hzh/openarmx_ws/src/openarmx_ros2/openarmx_bimanual_moveit_config
bash run_bimanual_moveit_sim.sh

# 2. 录制轨迹（新终端）
cd /home/hzh/openarmx_ws
source install/setup.bash
ros2 run openarmx_teach record_joint_states_always --rate 20

# 操作：
#   - 在RViz中用MoveIt拖动机械臂
#   - 按 空格键 开始录制
#   - 拖动机械臂执行动作
#   - 按 空格键 暂停录制
#   - 按 w 保存并退出

# 3. 压缩轨迹（去除停顿，保留0.5秒节奏）

ros2 run openarmx_teach compress_trajectory \
  joint_states_stream_*.yaml \
  my_trajectory.yaml \
  --pause-duration 0.5

# 4. 回放验证
ros2 run openarmx_teach play_joint_trajectory my_trajectory.yaml --all-joints
```

## 录制轨迹

### 基本用法

```bash
ros2 run openarmx_teach record_joint_states_always --rate 20
```

### 参数说明

- `--rate <Hz>`：采样频率（默认10Hz，推荐20Hz）
- `--outfile <name>`：自定义输出文件名（默认自动生成时间戳）

### 键盘控制

| 按键 | 功能 |
|------|------|
| `空格` / `p` | 开始/暂停录制 |
| `c` | 清空当前缓存 |
| `w` | 保存并退出 |
| `q` | 不保存退出 |

### 输出文件

- 默认文件名：`joint_states_stream_YYYYMMDD_HHMMSS.yaml`
- **默认保存位置**：`/home/hzh/openarmx_ws/src/openarmx_tools/openarmx_teach/trajectories/`
- 自定义保存路径：使用 `--outfile` 参数指定完整路径

```bash
# 保存到默认目录（推荐）
ros2 run openarmx_teach record_joint_states_always --rate 20

# 自定义保存路径
ros2 run openarmx_teach record_joint_states_always --rate 20 --outfile /path/to/custom.yaml
```

## 压缩轨迹

录制时会有**手动拖动的停顿**，导致轨迹时间很长且冗余。压缩工具可以：
- 自动检测并去除静止段
- 在动作间插入固定时长的停顿（保持节奏感）
- 大幅缩短总时长（通常可减少50-70%）

### 基本用法

```bash
# 完全去除停顿（最快）
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml

# 保留0.5秒停顿（推荐，有节奏感）
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 0.5

# 保留1秒停顿（更从容）
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 1.0
```

### 效果示例

以一个11.10秒的录制轨迹为例：

| 模式 | 命令 | 结果时长 | 说明 |
|------|------|----------|------|
| 原始 | - | 11.10s | 222个点，包含大量停顿 |
| 完全压缩 | `--pause-duration 0` | 5.29s | 68个点，无停顿 |
| 保留0.5s | `--pause-duration 0.5` | 7.29s | 68个点，4个0.5s停顿 |
| 保留1.0s | `--pause-duration 1.0` | 9.29s | 68个点，4个1.0s停顿 |

### 高级参数

```bash
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml \
  --threshold 0.01 \              # 位置变化阈值（弧度），越大越不敏感
  --min-static-frames 20 \        # 最少连续静止帧数
  --pause-duration 0.5 \          # 停顿时长（秒）
  --resample-dt 0.05              # 强制均匀重采样（可选）
```

## 回放轨迹

### 基本用法

```bash
# 回放所有关节（左右臂+夹爪）
ros2 run openarmx_teach play_joint_trajectory trajectory.yaml --all-joints

# 仅回放左臂
ros2 run openarmx_teach play_joint_trajectory trajectory.yaml --left-arm

# 仅回放右臂
ros2 run openarmx_teach play_joint_trajectory trajectory.yaml --right-arm

# 自定义速度（2倍速）
ros2 run openarmx_teach play_joint_trajectory trajectory.yaml --all-joints --rate-scale 2.0
```

### 回放参数

| 参数 | 说明 |
|------|------|
| `--all-joints` | 回放所有关节（推荐） |
| `--left-arm` | 仅左臂 |
| `--right-arm` | 仅右臂 |
| `--both-arms` | 双臂（不含夹爪） |
| `--rate-scale <float>` | 速度倍率（>1加速，<1减速） |
| `--sync-feedback` | 使用反馈同步夹爪 |
| `--sync-margin <sec>` | 反馈时间偏移 |

## 参数详解

### 压缩参数详解

#### `--threshold` (默认: 0.005)

位置变化阈值（弧度），用于判断是否在运动。

- **更大值** (如 0.01)：更不敏感，微小抖动会被忽略，压缩更激进
- **更小值** (如 0.001)：更敏感，保留更多细节，压缩更保守

```bash
# 忽略微小抖动，更激进压缩
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --threshold 0.01

# 保留更多细节
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --threshold 0.001
```

#### `--min-static-frames` (默认: 10)

最少连续静止帧数，低于此值的短暂停顿不会被压缩。

- **更大值** (如 20)：只有长时间停顿才被压缩，保守
- **更小值** (如 5)：短暂停顿也会被压缩，激进

```bash
# 只压缩长时间停顿（>1秒，20Hz采样）
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --min-static-frames 20

# 压缩短暂停顿（>0.25秒）
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --min-static-frames 5
```

#### `--pause-duration` (默认: 0.0)

在每个检测到的静止段后插入的停顿时长（秒）。

- **0.0**：完全去除停顿，动作连续
- **0.2-0.5**：轻微停顿，适合快速演示
- **0.5-1.0**：明显停顿，适合观察每个动作
- **>1.0**：长停顿，适合教学演示

```bash
# 不同节奏效果
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 0.0   # 连续
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 0.3   # 快节奏
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 0.5   # 标准节奏
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 1.0   # 慢节奏
```

#### `--resample-dt` (可选)

强制均匀重采样，所有点按固定时间间隔分布。

```bash
# 每0.05秒一个点
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --resample-dt 0.05
```

## 常见问题

### Q1: 压缩后动作太快？

**方案1**：增加停顿时长
```bash
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml --pause-duration 1.0
```

**方案2**：回放时降速
```bash
ros2 run openarmx_teach play_joint_trajectory output.yaml --all-joints --rate-scale 0.5
```

### Q2: 压缩后丢失了一些细节动作？

降低阈值或最小静止帧数：
```bash
ros2 run openarmx_teach compress_trajectory input.yaml output.yaml \
  --threshold 0.001 \
  --min-static-frames 5
```

### Q3: 录制时按空格没反应？

确保录制程序窗口处于活动状态（点击终端窗口）。

### Q4: 回放时报错 "Action server not available"？

确保仿真环境或实机控制器已启动：
```bash
# 检查控制器状态
ros2 action list | grep follow_joint_trajectory
```
