## 2. 控制频率优化

### 2.1 ros2_control频率

**配置文件:**
```yaml
# openarm_controllers.yaml

controller_manager:
  ros__parameters:
    update_rate: 200  # 从100提升到200 Hz
```

**推荐值:**
```
一般应用: 100 Hz
高性能:   200-300 Hz
极限:     500 Hz (需要实时系统)
```

**验证频率:**
```bash
# 监控实际更新频率
ros2 topic hz /joint_states

# 输出:
# average rate: 200.123
#   min: 0.004s max: 0.006s std dev: 0.00012s
```

### 2.2 CAN通信优化

**减少CAN消息数量:**
```python
# 不推荐:每个关节单独发送
for motor in motors:
    motor.set_position(...)
    time.sleep(0.001)  # 7个电机 = 7ms

# 推荐:批量发送(如果硬件支持)
# 或至少减少间隔
for motor in motors:
    motor.set_position(...)
# 无延迟,让CAN队列处理
```

**CAN总线利用率:**
```python
# 计算CAN带宽使用
# 每个MIT命令 ≈ 8字节数据 + 协议开销
# 1 Mbps CAN,理论最大约 10000帧/秒

# 14个电机 @ 100Hz = 1400帧/秒 (14%利用率) ✓
# 14个电机 @ 200Hz = 2800帧/秒 (28%利用率) ✓
# 14个电机 @ 500Hz = 7000帧/秒 (70%利用率) ⚠️
```

## 3. 实时性优化

### 3.1 Linux实时补丁

**安装RT-PREEMPT:**
```bash
# 检查当前内核
uname -r
# 6.8.0-60-generic

# 安装实时内核
sudo apt install linux-image-rt-amd64 linux-headers-rt-amd64

```

**验证实时内核:**
```bash
uname -r
# 应显示包含"rt"

cat /sys/kernel/realtime
# 1
```

### 3.2 CPU隔离

**隔离CPU核心:**
```bash
# 编辑GRUB配置
sudo vim /etc/default/grub

# 添加隔离参数
GRUB_CMDLINE_LINUX="isolcpus=2,3 nohz_full=2,3 rcu_nocbs=2,3"

# 更新GRUB
sudo update-grub
sudo reboot
```

**绑定进程到隔离核心:**
```bash
# 运行ros2_control在核心2
taskset -c 2 ros2 launch openarmx_bringup openarm.launch.py
```

### 3.3 进程优先级

**提高控制器优先级:**
```bash
# 查找controller_manager进程
ps aux | grep controller_manager

# 提高优先级(需要root)
sudo renice -20 -p <PID>

# 或设置实时调度
sudo chrt -f 80 -p <PID>
```

**Python脚本中设置:**
```python
import os

# 提高优先级
os.nice(-20)  # 需要权限

# 实时调度
import ctypes
libc = ctypes.CDLL('libc.so.6')
SCHED_FIFO = 1
libc.sched_setscheduler(0, SCHED_FIFO, ctypes.byref(ctypes.c_int(80)))
```

## 4. 代码优化

### 4.1 Python性能优化

**使用NumPy加速:**
```python
# 慢速:纯Python循环
positions = []
for motor in motors:
    fb = motor.get_feedback()
    positions.append(fb['position'])

# 快速:NumPy数组
import numpy as np
positions = np.array([motor.get_feedback()['position'] for motor in motors])
```

**避免重复计算:**
```python
# 不推荐:每次循环计算
while True:
    kp = calculate_kp()  # 重复计算
    motor.set_position(..., kp=kp, ...)

# 推荐:预先计算
kp = calculate_kp()
while True:
    motor.set_position(..., kp=kp, ...)
```

**使用Cython加速关键代码:**
```python
# critical_function.pyx
import numpy as np
cimport numpy as np

def fast_gravity_compensation(double[:] joint_angles):
    cdef int i
    cdef double[:] torques = np.zeros(7)

    for i in range(7):
        torques[i] = 2.0 * 9.8 * 0.3 * np.cos(joint_angles[i])

    return np.asarray(torques)

# 编译:
# cythonize -i critical_function.pyx
```

### 4.2 C++性能

**硬件接口优化:**
```cpp
// openarmx_hardware_interface.cpp

// 优化:避免重复创建对象
void OpenArmHardwareInterface::write(...)
{
    // 预分配数组
    static std::vector<double> kp_values(7, 20.0);
    static std::vector<double> kd_values(7, 1.0);

    // 批量处理
    for (size_t i = 0; i < hw_commands_positions_.size(); i++)
    {
        motors_[i].set_position(
            hw_commands_positions_[i],
            0.0,
            kp_values[i],
            kd_values[i],
            0.0
        );
    }
}
```

**内联函数:**
```cpp
// 频繁调用的小函数使用inline
inline double radToDeg(double rad) {
    return rad * 180.0 / M_PI;
}
```

### 7.3 自定义性能监控

```python
#!/usr/bin/env python3
"""
性能监控节点
"""
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import JointState
import time
import psutil

class PerformanceMonitor(Node):
    def __init__(self):
        super().__init__('performance_monitor')

        self.subscription = self.create_subscription(
            JointState,
            '/joint_states',
            self.callback,
            10
        )

        self.last_time = time.time()
        self.msg_count = 0

        # 定时报告
        self.timer = self.create_timer(5.0, self.report)

    def callback(self, msg):
        self.msg_count += 1

    def report(self):
        """性能报告"""
        current_time = time.time()
        dt = current_time - self.last_time

        # 消息频率
        freq = self.msg_count / dt

        # CPU使用率
        cpu_percent = psutil.cpu_percent(interval=1)

        # 内存使用
        mem = psutil.virtual_memory()

        print(f"========== 性能报告 ==========")
        print(f"消息频率: {freq:.2f} Hz")
        print(f"CPU使用: {cpu_percent:.1f}%")
        print(f"内存使用: {mem.percent:.1f}%")
        print(f"==============================")

        self.last_time = current_time
        self.msg_count = 0

def main():
    rclpy.init()
    monitor = PerformanceMonitor()
    rclpy.spin(monitor)
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## 8. 性能测试

### 8.1 控制延迟测试

```python
#!/usr/bin/env python3
"""
测试控制延迟
"""
import can
from robstride import Motor
import time
import numpy as np

bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motor = Motor(bus0, 1)
motor.enable()

latencies = []

for _ in range(100):
    # 发送命令
    start_time = time.time()

    motor.set_position(0.5, 0, kp=20, kd=1, torque=0)

    # 读取反馈
    fb = motor.get_feedback()

    end_time = time.time()

    latency = (end_time - start_time) * 1000  # ms
    latencies.append(latency)

    time.sleep(0.01)

latencies = np.array(latencies)

print(f"控制延迟统计:")
print(f"  平均: {np.mean(latencies):.2f} ms")
print(f"  最小: {np.min(latencies):.2f} ms")
print(f"  最大: {np.max(latencies):.2f} ms")
print(f"  标准差: {np.std(latencies):.2f} ms")

motor.disable()
```

### 8.2 同步性能测试

```python
#!/usr/bin/env python3
"""
测试多电机同步性能
"""
import can
from robstride import Motor
import time
import numpy as np

bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motors = [Motor(bus0, i) for i in range(1, 4)]

for m in motors:
    m.enable()

# 同步运动测试
target = 0.5
sync_errors = []

for _ in range(50):
    # 同时发送命令
    for motor in motors:
        motor.set_position(target, 0, kp=20, kd=1, torque=0)

    time.sleep(0.1)

    # 读取实际位置
    positions = [m.get_feedback()['position'] for m in motors]

    # 计算同步误差
    sync_error = np.max(positions) - np.min(positions)
    sync_errors.append(sync_error)

sync_errors = np.array(sync_errors)

print(f"同步误差统计:")
print(f"  平均: {np.mean(sync_errors):.4f} rad")
print(f"  最大: {np.max(sync_errors):.4f} rad")

for m in motors:
    m.disable()
```

## 10. 常见问题

### 10.1 控制频率达不到目标

**症状:** 实际频率低于配置值

**排查:**
```bash
# 1. 检查CPU使用率
htop

# 2. 检查实际频率
ros2 topic hz /joint_states

# 3. 检查调度延迟
cyclictest -p 80 -t5 -n
```

**解决:**
```
- 降低update_rate
- 使用实时内核
- 隔离CPU核心
- 优化代码
```

### 10.2 系统卡顿

**症状:** 周期性卡顿,控制不流畅

**原因:**
- 垃圾回收(Python)
- 系统服务干扰
- I/O阻塞

**解决:**
```python
# Python: 禁用GC(小心内存泄漏)
import gc
gc.disable()

# 定期手动GC
if iteration % 1000 == 0:
    gc.collect()
```

### 10.3 CAN总线拥塞

**症状:** 丢帧,通信超时

**排查:**
```bash
# 查看CAN错误
ip -s link show can0

# 输出:
# RX errors: 0
# TX errors: 0
# ...
```

**解决:**
```
- 降低控制频率
- 减少电机数量(分组控制)
- 优化CAN消息格式
- 检查线缆质量
```
