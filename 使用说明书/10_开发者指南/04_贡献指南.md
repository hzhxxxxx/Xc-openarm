# 04_贡献指南

## OpenArmX项目贡献指南

欢迎为OpenArmX项目做出贡献!本文档将指导您如何参与开发。

---

## 1. 贡献方式

### 1.1 可以贡献的内容

```
□ 代码贡献 - 新功能、Bug修复、性能优化
□ 文档贡献 - 教程、API文档、示例代码
□ 测试贡献 - 单元测试、集成测试、性能测试
□ 问题反馈 - Bug报告、功能请求
□ 代码审查 - Review Pull Request
□ 社区支持 - 回答问题、分享经验
```

### 1.2 贡献流程

```
1. Fork项目仓库
   ↓
2. 创建功能分支
   ↓
3. 开发并测试
   ↓
4. 提交Pull Request
   ↓
5. 代码审查
   ↓
6. 合并到主分支
```

**示例代码:**
```cpp
#include "openarmx_hardware/openarm_system.hpp"

#include <vector>
#include <memory>

namespace openarmx_hardware
{

OpenArmSystem::OpenArmSystem()
: hw_positions_(7, 0.0),
  hw_velocities_(7, 0.0),
  hw_efforts_(7, 0.0)
{
  // 初始化
}

hardware_interface::return_type OpenArmSystem::read(
  const rclcpp::Time & time,
  const rclcpp::Duration & period)
{
  // 实现
  for (size_t i = 0; i < hw_positions_.size(); ++i) {
    double position, velocity, torque;
    if (can0_->receive_motor_feedback(i + 1, position, velocity, torque)) {
      hw_positions_[i] = position;
      hw_velocities_[i] = velocity;
      hw_efforts_[i] = torque;
    }
  }

  return hardware_interface::return_type::OK;
}

}  // namespace openarmx_hardware
```

```cpp
// ✓ 使用ROS日志

// std::cout << "Error" << std::endl;  // 不要这样

#### 3.2.3 Docstring规范

**使用Google风格:**
```python
class Motor:
    """Robstride电机控制类

    此类封装了Robstride电机的CAN通信协议,提供了简洁的
    Python接口用于电机控制和状态读取。

    Attributes:
        bus: python-can的Bus对象
        motor_id: 电机ID (1-7)
        reduction_ratio: 减速比

    Example:
        >>> import can
        >>> bus = can.interface.Bus(channel='can0', bustype='socketcan')
        >>> motor = Motor(bus, motor_id=1)
        >>> motor.enable()
        >>> motor.set_position(position=0.5, kp=30.0)
    """

    def __init__(self, bus: can.interface.Bus, motor_id: int):
        """初始化电机对象

        Args:
            bus: CAN总线对象
            motor_id: 电机ID (1-7)

        Raises:
            ValueError: 如果motor_id超出范围
        """
        if not 1 <= motor_id <= 7:
            raise ValueError("motor_id must be 1-7")

        self.bus = bus
        self.motor_id = motor_id

### 4.2 提交规范

#### 4.2.1 提交消息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type类型:**
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档修改
- `style`: 代码格式(不影响功能)
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试
- `chore`: 构建/工具

**示例:**
```bash
feat(hardware): add force-torque sensor support

- Implement FTSensorInterface class
- Add URDF tags for FT sensor
- Create launch file for sensor node

Closes #123
```

```bash
fix(controller): resolve trajectory execution timeout

The controller was timing out on long trajectories due to
incorrect goal_time calculation. This fixes the issue by
properly computing goal_time based on trajectory duration.

Fixes #456
```

```bash
docs(tutorial): add visual servoing tutorial

- Create step-by-step tutorial
- Add Python example code
- Include troubleshooting section
```

#### 4.2.2 提交最佳实践

```bash
# ✓ 小而聚焦的提交
git add src/openarmx_hardware/src/ft_sensor.cpp
git add include/openarmx_hardware/ft_sensor.hpp
git commit -m "feat(hardware): add FT sensor interface"

# ✓ 相关文件一起提交
git add src/openarm_sensors/
git commit -m "feat(sensors): add RealSense camera support"

# ✗ 避免大而杂的提交
# git add .  # 不要这样
# git commit -m "various changes"  # 消息太模糊
```

## 7. 发布流程

### 7.1 版本号规范

**语义化版本 (Semantic Versioning):**
```
MAJOR.MINOR.PATCH

例如: 1.2.3
```

- **MAJOR**: 不兼容的API变更
- **MINOR**: 向后兼容的新功能
- **PATCH**: 向后兼容的Bug修复

### 7.2 发布检查清单

```
□ 所有测试通过
□ 文档已更新
□ CHANGELOG已更新
□ 版本号已更新(package.xml)
□ 创建Git tag
□ 构建二进制包
□ 更新发布说明
```
