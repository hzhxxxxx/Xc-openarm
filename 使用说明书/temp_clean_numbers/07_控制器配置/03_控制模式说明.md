# 控制模式说明

## MIT模式与CSP模式对比

深入了解Robstride电机的控制模式及其特点。


##  控制模式概述

### Robstride支持的模式

**Robstride电机支持多种控制模式:**
```
1. MIT模式 (Motion Control Mode)
   - 位置+速度+KP+KD+力矩混合控制
   - OpenArmX主要使用

2. CSP模式 (Cyclic Synchronous Position)
   - 周期同步位置模式
   - 纯位置控制

3. CSV模式 (Cyclic Synchronous Velocity)
   - 周期同步速度模式
   - 纯速度控制

4. CST模式 (Cyclic Synchronous Torque)
   - 周期同步力矩模式
   - 纯力矩控制
```

**OpenArmX使用:**
```
默认: MIT模式
原因:
  ✓ 灵活性高
  ✓ 支持柔顺控制
  ✓ 可调KP/KD
  ✓ 力矩前馈
```


##  MIT模式详解

### MIT模式原理

**控制方程:**
```
τ_command = KP × (θ_target - θ_actual)
          + KD × (ω_target - ω_actual)
          + τ_feedforward

τ_command: 命令力矩
θ_target: 目标位置
θ_actual: 实际位置
ω_target: 目标速度
ω_actual: 实际速度
τ_feedforward: 前馈力矩
```

**命令格式:**
```python
motor.set_position(
    position=0.5,    # θ_target (rad)
    velocity=0.0,    # ω_target (rad/s)
    kp=20.0,         # 位置增益
    kd=1.0,          # 速度增益
    torque=0.0       # τ_feedforward (Nm)
)
```

### MIT模式特点

**优点:**
```
✓ 柔顺控制
  - 低KP/KD实现柔性
  - 示教模式易推动
  - 人机协作安全

✓ 力矩控制
  - 可叠加力矩
  - 重力补偿
  - 接触力控制

✓ 参数可调
  - 实时调整KP/KD
  - 适应不同任务
  - 动态刚度调节

✓ 混合控制
  - 位置+力矩
  - 速度前馈
  - 阻抗控制
```

**缺点:**
```
✗ 参数调试复杂
  - 需要调KP/KD
  - 不同负载需调整

✗ 精度依赖参数
  - 高精度需高增益
  - 可能引起震荡
```

### MIT模式应用

**场景1: 示教录制**
```python
# 低刚度,易推动
motor.set_position(
    position=current_position,  # 跟随当前位置
    velocity=0.0,
    kp=2.0,      # 低KP
    kd=0.2,      # 低KD
    torque=0.0
)
```

**场景2: 轨迹跟踪**
```python
# 中等刚度,平衡性能
motor.set_position(
    position=trajectory_point,
    velocity=0.0,
    kp=20.0,     # 中等KP
    kd=1.0,      # 中等KD
    torque=0.0
)
```

**场景3: 精确定位**
```python
# 高刚度,高精度
motor.set_position(
    position=target_position,
    velocity=0.0,
    kp=50.0,     # 高KP
    kd=2.5,      # 高KD
    torque=0.0
)
```

**场景4: 重力补偿**
```python
# 计算重力力矩
gravity_torque = calculate_gravity_torque(joint_positions)

motor.set_position(
    position=target_position,
    velocity=0.0,
    kp=15.0,
    kd=1.0,
    torque=gravity_torque  # 补偿重力
)
```

**场景5: 力控抓取**
```python
# 柔顺抓取
motor.set_position(
    position=grasp_position,
    velocity=0.0,
    kp=5.0,      # 低KP,柔顺
    kd=0.5,
    torque=1.0   # 小力矩保持
)
```


##  CSP模式详解

### CSP模式原理

**控制方式:**
```
用户: 发送目标位置
电机: 内部PID跟踪位置
特点: 纯位置控制,内部闭环
```

**命令格式:**
```python
# 伪代码 (Robstride SDK可能不同)
motor.set_mode(CSP_MODE)
motor.set_target_position(0.5)  # rad
```

### CSP模式特点

**优点:**
```
✓ 简单易用
  - 只需发送位置
  - 无需调KP/KD
  - 参数固定

✓ 精度稳定
  - 出厂调优
  - 一致性好
  - 适合批量部署

✓ 标准化
  - CANopen标准
  - 兼容性好
```

**缺点:**
```
✗ 刚度固定
  - 无法调节柔顺性
  - 不适合示教
  - 人机交互差

✗ 无力矩控制
  - 无法叠加力矩
  - 难以重力补偿
  - 力控困难

✗ 灵活性低
  - 参数不可调
  - 难以优化特定任务
```

### CSP模式应用

**适用场景:**
```
- 固定任务,无需调节
- 高精度重复定位
- 标准化应用
- 无力控需求
```

**不适用场景:**
```
- 示教录制
- 人机协作
- 力控应用
- 动态调节刚度
```


##  MIT vs CSP对比

### 功能对比表

| 特性 | MIT模式 | CSP模式 |
|------|---------|---------|
| **控制方式** | 位置+速度+KP+KD+力矩 | 纯位置 |
| **参数调节** | 实时可调KP/KD | 固定 |
| **柔顺控制** | ✓ 支持 | ✗ 不支持 |
| **力矩前馈** | ✓ 支持 | ✗ 不支持 |
| **示教模式** | ✓ 易实现 | ✗ 困难 |
| **精度** | 依赖参数 | 稳定 |
| **复杂度** | ★★★★☆ | ★★☆☆☆ |
| **灵活性** | ★★★★★ | ★★☆☆☆ |
| **人机协作** | ★★★★★ | ★☆☆☆☆ |

### 性能对比

**响应速度:**
```
MIT模式: 取决于KP/KD设置
  - 高KP → 快速响应
  - 低KP → 慢速响应

CSP模式: 固定响应速度
  - 出厂优化
  - 中等速度
```

**定位精度:**
```
MIT模式: 可调
  - 高KP → 高精度
  - 低KP → 低精度

CSP模式: 固定高精度
  - ±0.01 rad 典型值
```

**能耗:**
```
MIT模式: 可优化
  - 低KP → 低能耗
  - 高KP → 高能耗

CSP模式: 中等能耗
  - 固定增益
```

### 应用场景对比

**MIT模式适用:**
```
✓ 示教录制
✓ 人机协作
✓ 力控装配
✓ 柔顺抓取
✓ 动态负载
✓ 研发测试
```

**CSP模式适用:**
```
✓ 固定自动化
✓ 高精度定位
✓ 批量生产
✓ 简单应用
✓ 标准设备集成
```


##  其他控制模式

### CSV模式 (速度控制)

**原理:**
```
用户: 发送目标速度
电机: 内部PID跟踪速度
```

**应用:**
```
- 传送带
- 连续旋转
- 速度同步
```

**示例:**
```python
motor.set_mode(CSV_MODE)
motor.set_target_velocity(1.0)  # rad/s
```

### CST模式 (力矩控制)

**原理:**
```
用户: 发送目标力矩
电机: 直接输出力矩
```

**应用:**
```
- 纯力控
- 阻抗控制
- 力传感器标定
```

**示例:**
```python
motor.set_mode(CST_MODE)
motor.set_target_torque(2.0)  # Nm
```

**⚠️ 注意:**
```
纯力矩控制无位置限制
需要外部安全监控
避免超限和碰撞
```

### 混合模式

**MIT本质是混合模式:**
```
位置控制 + 速度控制 + 力矩控制

可实现:
- 位置主导 (高KP, 低torque)
- 力矩主导 (低KP, 高torque)
- 阻抗控制 (中KP, 中torque)
```


##  OpenArmX控制模式配置

### 当前配置

**默认使用MIT模式:**
```cpp
// openarmx_hardware_interface.cpp

void OpenArmHardwareInterface::write(...)
{
    // MIT模式控制
    motors_[i].set_position(
        target_position,
        0.0,      // velocity
        kp,       // 可配置
        kd,       // 可配置
        0.0       // torque
    );
}
```

### 切换到CSP模式

**修改硬件接口:**
```cpp
// 初始化时设置模式
void OpenArmHardwareInterface::on_activate(...)
{
    for (auto& motor : motors_)
    {
        // motor.set_mode(CSP_MODE);  // 如果SDK支持
    }
}

// 控制时仅发送位置
void OpenArmHardwareInterface::write(...)
{
    // motors_[i].set_target_position(target_position);
}
```

**⚠️ 注意:**
```
切换模式可能需要:
1. SDK支持
2. 固件版本
3. 重新编译
4. 测试验证
```


##  模式选择指南

### 决策流程图

```
需要示教录制?
  ├─ 是 → MIT模式
  └─ 否
      ↓
需要力控?
  ├─ 是 → MIT模式
  └─ 否
      ↓
需要调节刚度?
  ├─ 是 → MIT模式
  └─ 否
      ↓
固定任务,高精度?
  ├─ 是 → CSP模式
  └─ 否 → MIT模式 (更灵活)
```

### 推荐配置

**OpenArmX推荐:**
```
MIT模式 (默认)

原因:
✓ 支持示教
✓ 柔顺控制
✓ 灵活性高
✓ 适合研发和多样化应用
```

**特殊应用:**
```
如果是固定生产线,无需示教:
  → 可考虑CSP模式

如果需要纯力控:
  → MIT模式 (低KP + 力矩)
  或 CST模式
```


##  实际应用案例

### 案例1: 柔顺装配 (MIT)

```python
#!/usr/bin/env python3
"""
柔顺装配示例 - 使用MIT模式
"""
import can
from robstride import Motor
import time

bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motor = Motor(bus0, 4)  # 肘部关节
motor.enable()

# 阶段1: 移动到插孔上方 (高刚度)
print("阶段1: 快速定位")
motor.set_position(
    position=-0.8,
    velocity=0.0,
    kp=30.0,     # 高KP,快速
    kd=1.5,
    torque=0.0
)
time.sleep(3)

# 阶段2: 接近插孔 (中刚度)
print("阶段2: 接近目标")
motor.set_position(
    position=-1.0,
    velocity=0.0,
    kp=15.0,     # 中KP
    kd=1.0,
    torque=0.0
)
time.sleep(2)

# 阶段3: 插入 (低刚度 + 力控)
print("阶段3: 柔顺插入")
for i in range(20):
    motor.set_position(
        position=-1.0,
        velocity=0.0,
        kp=5.0,      # 低KP,柔顺
        kd=0.5,
        torque=2.0   # 小力矩推入
    )
    time.sleep(0.1)

    # 检测接触力
    fb = motor.get_feedback()
    if abs(fb['torque']) > 5.0:
        print("检测到大阻力,停止")
        break

print("装配完成")
motor.disable()
```

### 案例2: 示教录制 (MIT)

```python
#!/usr/bin/env python3
"""
示教录制 - MIT模式低刚度
"""
import can
from robstride import Motor
import time

bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motors = [Motor(bus0, i) for i in range(1, 8)]

for m in motors:
    m.enable()

print("进入示教模式,可以手动拖动机械臂")
print("按Ctrl+C退出")

trajectory = []

try:
    while True:
        # 读取当前位置
        positions = [m.get_feedback()['position'] for m in motors]

        # 记录轨迹
        trajectory.append(positions)

        # 低刚度跟随
        for i, motor in enumerate(motors):
            if i < 3:  # 肩部
                kp, kd = 3.0, 0.3
            else:      # 肘腕
                kp, kd = 2.0, 0.2

            motor.set_position(
                position=positions[i],  # 跟随当前位置
                velocity=0.0,
                kp=kp,    # 低刚度,易推动
                kd=kd,
                torque=0.0
            )

        time.sleep(0.1)

except KeyboardInterrupt:
    print(f"\n录制结束,共{len(trajectory)}个点")

    # 保存轨迹
    import yaml
    with open('taught_trajectory.yaml', 'w') as f:
        yaml.dump({'points': trajectory}, f)

    for m in motors:
        m.disable()
```

### 案例3: 高精度定位 (CSP或高增益MIT)

```python
#!/usr/bin/env python3
"""
高精度定位 - 使用高增益MIT
(如果使用CSP,则无需KP/KD)
"""
import can
from robstride import Motor
import time

bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motor = Motor(bus0, 1)
motor.enable()

# 精确目标位置
target_positions = [0.0, 0.5, 1.0, 0.5, 0.0]

for target in target_positions:
    print(f"移动到 {target:.3f} rad")

    # MIT高增益模式
    motor.set_position(
        position=target,
        velocity=0.0,
        kp=80.0,     # 高KP,高精度
        kd=3.0,      # 高KD,无震荡
        torque=0.0
    )

    # 等待到达
    time.sleep(3)

    # 验证精度
    fb = motor.get_feedback()
    error = abs(fb['position'] - target)
    print(f"  实际: {fb['position']:.4f}, 误差: {error:.4f} rad")

motor.disable()
```


##  常见问题

### MIT模式调参困难

**问题:** 不同任务需要不同参数,难以管理

**解决:**
```python
# 使用参数字典
PARAM_PROFILES = {
    'teaching': {'kp': 3.0, 'kd': 0.3},
    'normal':   {'kp': 20.0, 'kd': 1.0},
    'precise':  {'kp': 50.0, 'kd': 2.5},
    'compliant':{'kp': 5.0, 'kd': 0.5}
}

# 切换配置
profile = PARAM_PROFILES['teaching']
motor.set_position(pos, 0, profile['kp'], profile['kd'], 0)
```

### CSP模式缺少灵活性

**问题:** CSP模式无法调节,太刚

**解决:**
```
使用MIT模式代替
设置固定的中等增益 (KP=20, KD=1.0)
获得CSP的简单性 + MIT的灵活性
```

### 力矩控制不稳定

**问题:** 使用MIT力矩前馈时不稳定

**解决:**
```
1. 降低KP/KD,避免与力矩冲突
2. 力矩值从小到大逐步增加
3. 添加力矩限幅
4. 监控实际力矩,避免超限
```
