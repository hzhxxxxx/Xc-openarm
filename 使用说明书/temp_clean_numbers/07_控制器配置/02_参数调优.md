# 参数调优

## 控制器KP/KD参数调优指南

掌握控制参数调节,优化机器人运动性能。


##  参数调优概述

### 什么是KP/KD

**PD控制器原理:**
```
控制输出 = KP × 位置误差 + KD × 速度误差

KP (比例增益):
  - 响应位置误差
  - 值越大,响应越快
  - 过大会震荡

KD (微分增益):
  - 响应速度误差
  - 提供阻尼
  - 抑制震荡
```

**物理意义:**
```
KP: 类似弹簧刚度
  - 高KP = 硬弹簧 = 快速但可能震荡
  - 低KP = 软弹簧 = 慢速但稳定

KD: 类似阻尼器
  - 高KD = 强阻尼 = 平滑但慢
  - 低KD = 弱阻尼 = 快但可能震荡
```

### OpenArmX控制架构

```
ros2_control (位置目标)
    ↓
Hardware Interface (位置 → KP/KD/力矩)
    ↓
Robstride电机MIT模式 (位置+速度+KP+KD+力矩)
    ↓
电机内部控制器
    ↓
实际运动
```

**MIT模式命令:**
```python
motor.set_position(
    position=0.5,    # 目标位置 (rad)
    velocity=0.0,    # 前馈速度 (rad/s)
    kp=20.0,         # 位置增益
    kd=1.0,          # 速度增益
    torque=0.0       # 前馈力矩 (Nm)
)
```

##  参数调优方法

### 经验调参法

**步骤:**

```
1. 从保守值开始
   KP = 10, KD = 0.5

2. 逐步增加KP
   观察响应速度
   直到出现轻微震荡

3. 增加KD抑制震荡
   调整到运动平滑

4. 微调优化
   平衡速度和稳定性

5. 负载测试
   加载后可能需要增加KP
```

**调参口诀:**
```
KP调速度,KD调平滑
先调KP后调KD
震荡就加KD
慢了就加KP
```

### Ziegler-Nichols法

**理论方法:**

```
1. 设置KD = 0

2. 逐步增加KP直到系统震荡
   记录临界KP值: Ku
   记录震荡周期: Tu

3. 计算PD参数:
   KP = 0.8 × Ku
   KD = 0.1 × Ku × Tu
```

**示例:**
```
临界值: Ku = 100, Tu = 0.2s

计算:
KP = 0.8 × 100 = 80
KD = 0.1 × 100 × 0.2 = 2.0

起始值: KP=80, KD=2.0
微调优化
```

**测试设置:**
```python
#!/usr/bin/env python3
"""
参数调优测试脚本
"""
import can
from robstride import Motor
import time

bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motor = Motor(bus0, 1)  # 测试电机1

motor.enable()
time.sleep(1)

# 参数范围
kp_values = [10, 20, 30, 50, 80]
kd_values = [0.5, 1.0, 1.5, 2.0, 3.0]

# 测试目标
target_position = 0.5  # rad

for kp in kp_values:
    for kd in kd_values:
        print(f"\n测试: KP={kp}, KD={kd}")

        # 回零
        motor.set_position(0.0, vel=0.0, kp=20, kd=1.0, torque=0.0)
        time.sleep(2)

        # 移动到目标
        start_time = time.time()
        motor.set_position(target_position, vel=0.0, kp=kp, kd=kd, torque=0.0)

        # 记录响应
        positions = []
        times = []

        for i in range(50):  # 5秒,10Hz采样
            fb = motor.get_feedback()
            positions.append(fb['position'])
            times.append(time.time() - start_time)
            time.sleep(0.1)

        # 分析性能
        # 超调量,稳定时间,震荡等
        overshoot = max(positions) - target_position
        settling_time = next((t for t, p in zip(times, positions)
                             if abs(p - target_position) < 0.02), times[-1])

        print(f"  超调: {overshoot:.4f} rad")
        print(f"  稳定时间: {settling_time:.2f} s")

        # 保存数据用于后续分析
        # ...

motor.disable()
```

**性能指标:**
```
□ 上升时间: 从10%到90%所需时间
□ 超调量: 超过目标的最大偏差
□ 稳定时间: 到达±2%误差带所需时间
□ 震荡次数: 震荡周期数
```

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from rcl_interfaces.srv import SetParameters
from rcl_interfaces.msg import Parameter, ParameterValue, ParameterType

class ParameterTuner(Node):
    def __init__(self):
        super().__init__('parameter_tuner')

        self.client = self.create_client(
            SetParameters,
            '/hardware_interface/set_parameters'
        )

    def set_kp_kd(self, joint_name, kp, kd):
        """动态设置KP/KD"""
        request = SetParameters.Request()

        # KP参数
        kp_param = Parameter()
        kp_param.name = f'{joint_name}.kp'
        kp_param.value.type = ParameterType.PARAMETER_DOUBLE
        kp_param.value.double_value = kp

        # KD参数
        kd_param = Parameter()
        kd_param.name = f'{joint_name}.kd'
        kd_param.value.type = ParameterType.PARAMETER_DOUBLE
        kd_param.value.double_value = kd

        request.parameters = [kp_param, kd_param]

        future = self.client.call_async(request)
        rclpy.spin_until_future_complete(self, future)

        self.get_logger().info(f'设置 {joint_name}: KP={kp}, KD={kd}')

def main():
    rclpy.init()

    tuner = ParameterTuner()

    # 动态调整left_joint_1的参数
    tuner.set_kp_kd('left_joint_1', kp=30.0, kd=1.5)

    rclpy.shutdown()

if __name__ == '__main__':
    main()
```


##  调优案例

### 案例1: 示教模式参数

**需求:** 手动拖动机械臂,阻力小

**调整前:**
```
KP = 20, KD = 1.0
现象: 难以推动,阻力大
```

**调整后:**
```
KP = 2, KD = 0.2
现象: 易于推动,但会下垂
```

**优化:**
```
KP = 3, KD = 0.3
+ 重力补偿力矩

结果: 易推动且不下垂
```

### 案例2: 高速运动震荡

**需求:** 快速运动,出现震荡

**现象:**
```
KP = 50, KD = 1.0
到达目标后震荡5-6次才稳定
```

**分析:**
```
KP过高,KD不足以抑制震荡
```

**调整:**
```
方案1: 降低KP
  KP = 30, KD = 1.0
  结果: 震荡减少,但速度变慢

方案2: 增加KD
  KP = 50, KD = 2.5
  结果: 震荡消失,速度保持

选择方案2
```

### 案例3: 负载运动下垂

**需求:** 抓取重物后,手臂下垂

**现象:**
```
KP = 20, KD = 1.0
抓取1kg物体,手臂下降5cm
```

**调整:**
```
方案1: 增加KP
  KP = 40, KD = 1.0
  结果: 下垂减少,但震荡

方案2: KP+KD同时增加
  KP = 35, KD = 1.8
  结果: 下垂消失,运动平滑

方案3: 添加重力补偿
  KP = 25, KD = 1.2 + 重力补偿
  结果: 最优,能耗低

选择方案3
```

### 案例4: 精确定位

**需求:** 插孔任务,定位精度±0.5mm

**调整前:**
```
KP = 20, KD = 1.0
定位误差: ±2mm
```

**调整过程:**
```
尝试1: KP = 50, KD = 1.0
  误差: ±1.5mm, 有震荡

尝试2: KP = 50, KD = 2.0
  误差: ±1mm, 无震荡

尝试3: KP = 80, KD = 3.0
  误差: ±0.4mm, 无震荡
  ✓ 满足要求
```

##  调参工具

### 自动调参脚本

```python
#!/usr/bin/env python3
"""
自动KP/KD调参工具
"""
import can
from robstride import Motor
import time
import numpy as np

class AutoTuner:
    def __init__(self, motor):
        self.motor = motor

    def evaluate_performance(self, kp, kd, target_pos):
        """
        评估性能
        返回分数 (越高越好)
        """
        # 回零
        self.motor.set_position(0.0, 0.0, 20, 1.0, 0.0)
        time.sleep(2)

        # 运动到目标
        start_time = time.time()
        self.motor.set_position(target_pos, 0.0, kp, kd, 0.0)

        # 记录响应
        positions = []
        times = []

        for _ in range(50):
            fb = self.motor.get_feedback()
            positions.append(fb['position'])
            times.append(time.time() - start_time)
            time.sleep(0.1)

        # 计算指标
        positions = np.array(positions)
        error = np.abs(positions - target_pos)

        # 稳定时间 (到达±2%误差带)
        settling_threshold = 0.02
        settling_idx = np.where(error < settling_threshold)[0]
        settling_time = times[settling_idx[0]] if len(settling_idx) > 0 else times[-1]

        # 超调量
        overshoot = np.max(positions) - target_pos

        # 最终误差
        final_error = error[-10:].mean()

        # 综合评分 (越小越好,转为越大越好)
        score = 1.0 / (settling_time + 10*abs(overshoot) + 100*final_error + 0.1)

        print(f"KP={kp:.1f}, KD={kd:.2f} → "
              f"稳定时间={settling_time:.2f}s, "
              f"超调={overshoot:.4f}, "
              f"误差={final_error:.4f}, "
              f"得分={score:.3f}")

        return score

    def grid_search(self, kp_range, kd_range, target_pos=0.5):
        """网格搜索最优参数"""
        best_score = -1
        best_kp = None
        best_kd = None

        for kp in kp_range:
            for kd in kd_range:
                score = self.evaluate_performance(kp, kd, target_pos)

                if score > best_score:
                    best_score = score
                    best_kp = kp
                    best_kd = kd

        print(f"\n最优参数: KP={best_kp}, KD={best_kd}, 得分={best_score:.3f}")
        return best_kp, best_kd

# 使用
bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
motor = Motor(bus0, 1)
motor.enable()

tuner = AutoTuner(motor)

# 搜索范围
kp_range = [10, 15, 20, 25, 30, 40, 50]
kd_range = [0.5, 0.8, 1.0, 1.2, 1.5, 2.0, 2.5]

best_kp, best_kd = tuner.grid_search(kp_range, kd_range)

motor.disable()
```

```python
#!/usr/bin/env python3
"""
实时调参GUI (使用tkinter)
"""
import tkinter as tk
from tkinter import Scale
import can
from robstride import Motor
import threading

class TunerGUI:
    def __init__(self):
        self.window = tk.Tk()
        self.window.title("KP/KD调参工具")

        # 初始化电机
        bus0 = can.interface.Bus(channel='can0', bustype='socketcan')
        self.motor = Motor(bus0, 1)
        self.motor.enable()

        # KP滑块
        tk.Label(self.window, text="KP:").pack()
        self.kp_scale = Scale(self.window, from_=0, to=100, resolution=1,
                             orient=tk.HORIZONTAL, length=400,
                             command=self.update_params)
        self.kp_scale.set(20)
        self.kp_scale.pack()

        # KD滑块
        tk.Label(self.window, text="KD:").pack()
        self.kd_scale = Scale(self.window, from_=0, to=5, resolution=0.1,
                             orient=tk.HORIZONTAL, length=400,
                             command=self.update_params)
        self.kd_scale.set(1.0)
        self.kd_scale.pack()

        # 目标位置
        tk.Label(self.window, text="目标位置 (rad):").pack()
        self.target_scale = Scale(self.window, from_=-3.14, to=3.14, resolution=0.01,
                                 orient=tk.HORIZONTAL, length=400)
        self.target_scale.set(0.5)
        self.target_scale.pack()

        # 执行按钮
        tk.Button(self.window, text="执行运动", command=self.execute_motion).pack()

        # 回零按钮
        tk.Button(self.window, text="回零", command=self.go_home).pack()

        # 状态显示
        self.status_label = tk.Label(self.window, text="就绪")
        self.status_label.pack()

    def update_params(self, event=None):
        """更新参数显示"""
        kp = self.kp_scale.get()
        kd = self.kd_scale.get()
        self.status_label.config(text=f"当前参数: KP={kp}, KD={kd}")

    def execute_motion(self):
        """执行运动"""
        kp = self.kp_scale.get()
        kd = self.kd_scale.get()
        target = self.target_scale.get()

        self.status_label.config(text=f"执行: 目标={target:.2f}, KP={kp}, KD={kd}")

        # 在新线程中执行,避免阻塞GUI
        threading.Thread(target=self._move, args=(target, kp, kd)).start()

    def _move(self, target, kp, kd):
        self.motor.set_position(target, 0.0, kp, kd, 0.0)

    def go_home(self):
        """回零"""
        self.motor.set_position(0.0, 0.0, 20, 1.0, 0.0)
        self.status_label.config(text="回零中...")

    def run(self):
        self.window.mainloop()
        self.motor.disable()

# 运行
gui = TunerGUI()
gui.run()
```


##  常见问题

### 震荡问题

**症状:** 到达目标后持续震荡

**原因:** KP过高,KD不足

**解决:**
```
1. 降低KP: 减少20-30%
2. 增加KD: 增加50-100%
3. 检查机械松动
4. 降低控制频率
```

### 响应慢

**症状:** 移动速度慢,响应迟缓

**原因:** KP过低

**解决:**
```
1. 增加KP: 增加50-100%
2. 适当增加KD (保持平滑)
3. 检查负载是否过重
```

### 稳态误差

**症状:** 无法准确到达目标位置

**原因:** KP不足或有负载

**解决:**
```
1. 增加KP
2. 添加积分项 (KI)
3. 添加重力/摩擦力补偿
```

### 噪声/抖动

**症状:** 微小快速震动

**原因:** KD过高或传感器噪声

**解决:**
```
1. 降低KD
2. 添加低通滤波
3. 降低控制频率
```
