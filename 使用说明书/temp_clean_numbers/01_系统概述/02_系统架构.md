# 系统架构

## OpenArmX 系统架构详解

本文档详细介绍 OpenArmX 双臂机器人的软硬件架构、技术栈和控制流程。

**自由度(DOF)配置:**
- 左臂: 7 DOF(关节1-7)
- 右臂: 7 DOF(关节1-7)
- 左夹爪: 1 DOF
- 右夹爪: 1 DOF
- **总计: 16 DOF**

### 话题(Topic)架构

**关键话题列表:**

```
/joint_states (sensor_msgs/JointState)
├─ 发布者: ros2_control_node
└─ 订阅者: robot_state_publisher, move_group

/tf (tf2_msgs/TFMessage)
├─ 发布者: robot_state_publisher
└─ 订阅者: move_group, rviz2

/left_arm_controller/joint_trajectory (trajectory_msgs/JointTrajectory)
├─ 发布者: move_group
└─ 订阅者: left_arm_controller

/right_arm_controller/joint_trajectory (trajectory_msgs/JointTrajectory)
├─ 发布者: move_group
└─ 订阅者: right_arm_controller

/left_gripper_controller/commands (std_msgs/Float64MultiArray)
├─ 发布者: 用户应用
└─ 订阅者: left_gripper_controller

/right_gripper_controller/commands (std_msgs/Float64MultiArray)
├─ 发布者: 用户应用
└─ 订阅者: right_gripper_controller
```

##  数据流

### 读取流程(从硬件到软件)

```
┌────────────┐
│ 电机       │
│ (实际位置)  │
└─────┬──────┘
      │ CAN帧
      ▼
┌────────────────────┐
│ CAN驱动(SocketCAN) │
└─────┬──────────────┘
      │ can_frame
      ▼
┌────────────────────────┐
│ RobstrideMotor::read() │
│ (解码位置/速度/力矩)    │
└─────┬──────────────────┘
      │ motor_state
      ▼
┌─────────────────────────────────┐
│ OpenArmV10Hardware::read()      │
│ (更新 hw_states_position_)      │
└─────┬───────────────────────────┘
      │ joint_states
      ▼
┌──────────────────────────┐
│ ros2_control Framework   │
│ (更新控制器状态)          │
└─────┬────────────────────┘
      │ /joint_states 话题
      ▼
┌──────────────────────┐
│ robot_state_publisher│
│ (发布TF树)            │
└─────┬────────────────┘
      │ /tf 话题
      ▼
┌──────────────────┐
│ MoveIt / RViz    │
│ (可视化和规划)    │
└──────────────────┘
```

**更新频率:**
- CAN读取: ~1000 Hz(电机内部)
- Hardware Interface read(): 100 Hz(配置值)
- joint_states发布: 100 Hz
- TF发布: 100 Hz

### 写入流程(从软件到硬件)

**轨迹执行流程:**

```
┌──────────────────┐
│ MoveIt           │
│ (规划轨迹)        │
└─────┬────────────┘
      │ FollowJointTrajectory Action
      ▼
┌───────────────────────────────┐
│ JointTrajectoryController     │
│  (插值生成关节命令)             │
└─────┬─────────────────────────┘
      │ joint_position_command (100Hz)
      ▼
┌─────────────────────────────────┐
│ OpenArmV10Hardware::write()     │
│ (读取 hw_commands_position_)    │
└─────┬───────────────────────────┘
      │ target_position
      ▼
┌──────────────────────────────────┐
│ RobstrideMotor::setCommand()     │
│  (编码MIT模式控制帧)               │
└─────┬────────────────────────────┘
      │ CAN帧
      ▼
┌────────────────────┐
│ CAN驱动(SocketCAN) │
└─────┬──────────────┘
      │
      ▼
┌────────────┐
│ 电机       │
│ (执行运动)  │
└────────────┘
```

### 完整控制循环

**100Hz控制循环:**

```
     ┌─────────────────────────────────────┐
     │                                     │
     │    每10ms执行一次                   │
     │                                     │
     ▼                                     │
┌─────────┐       ┌─────────┐       ┌──────────┐
│  Read   │──────►│ Update  │──────►│  Write   │
│ (读传感器)│       │ (控制器) │       │ (写命令)  │
└─────────┘       └─────────┘       └──────────┘
     │                  │                  │
     │                  │                  │
     ▼                  ▼                  ▼
  获取电机          计算控制律          发送电机
  位置/速度          (PID/轨迹)          目标位置
  力矩反馈          插值等              KP/KD等
     │                  │                  │
     └──────────────────┴──────────────────┘
                        │
                        │ 10ms后
                        └──────────────────┐
                                          │
                                          ▼
                                      下一循环
```

##  通信协议

### CAN 协议

**Robstride MIT模式控制帧:**

```
CAN ID: 0x000 - 0x01F (电机ID: 1-16)

数据帧格式(8字节):
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│Byte 0│Byte 1│Byte 2│Byte 3│Byte 4│Byte 5│Byte 6│Byte 7│
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ Pos_H│ Pos_L│ Vel_H│ Vel_L│  KP_H│  KP_L│  KD_H│  KD_L│
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘

Pos: 位置 (-12.5 ~ 12.5 rad, 16bit)
Vel: 速度 (-65 ~ 65 rad/s, 16bit)
KP:  位置增益 (0 ~ 500, 16bit)
KD:  速度增益 (0 ~ 5, 16bit)
```

**电机反馈帧:**

```
数据帧格式(8字节):
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│Byte 0│Byte 1│Byte 2│Byte 3│Byte 4│Byte 5│Byte 6│Byte 7│
├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ Pos_H│ Pos_L│ Vel_H│ Vel_L│ Tor_H│ Tor_L│ Temp │ Error│
└──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘

Pos:  当前位置 (rad)
Vel:  当前速度 (rad/s)
Tor:  当前力矩 (Nm)
Temp: 温度 (°C)
Error: 错误代码
```

##  总结

OpenArmX 采用**分层模块化架构**,各层职责清晰:

1. **硬件层**: Robstride电机提供高性能执行
2. **驱动层**: CAN通信库封装底层协议
3. **接口层**: ros2_control标准化硬件接口
4. **控制层**: 多种控制器支持不同应用
5. **规划层**: MoveIt提供智能运动规划
6. **应用层**: 录制回放、示教等高级功能

这种架构具有以下优势:
- ✅ **标准化**: 符合ROS-Industrial规范
- ✅ **可扩展**: 易于添加新功能
- ✅ **可维护**: 模块独立,便于调试
- ✅ **高性能**: 100Hz控制频率,低延迟
