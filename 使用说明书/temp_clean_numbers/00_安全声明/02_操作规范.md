# 操作规范

## 正确操作流程与规范

本文档规定了 OpenArmX 机器人系统的标准操作流程，所有操作人员必须严格遵守。


##  操作人员要求

### 资质要求
- ✅ 具备**机器人或自动化系统**相关知识
- ✅ 熟悉**ROS2 和 Linux 系统**基本操作
- ✅ 了解**电气安全**和机械安全知识
- ✅ 完成**OpenArmX 系统培训**
- ✅ 阅读并理解**完整使用说明书**

### 健康要求
- ✅ 身体健康，反应灵敏
- ✅ 无色盲，视力良好
- ❌ 不得在疲劳、饮酒或服用影响判断力的药物后操作

### 着装要求
- ✅ 穿戴**防护服**和**防护眼镜**（推荐）
- ✅ 穿**防滑、合脚的鞋子**
- ❌ **不得**穿宽松衣物，避免被机器人夹住
- ❌ **不得**佩戴长项链、手环等饰品
- ❌ **不得**留长发散披，需束起

##  标准开机流程

### 步骤1：系统准备
```bash
#  启动控制计算机
#  打开终端，进入工作空间
cd ~/openarmx_robotstride_ws-cc

#  加载 ROS2 环境
source /opt/ros/humble/setup.bash
source install/setup.bash
```

### 步骤2：启动CAN接口
```bash
# 使用 Python 脚本启动 CAN（推荐）
python3 src/motor_tests_openarmx_com/en_all_can.py

# 或使用手动命令
sudo ip link set can0 type can bitrate 1000000
sudo ip link set can0 up
sudo ip link set can1 type can bitrate 1000000
sudo ip link set can1 up
```

**检查点：**
- CAN接口LED指示灯应常亮
- 执行 `ip link show can0` 应显示状态为 UP

### 步骤3：上电
1. **确认人员已撤离工作区域**
2. **手悬停在急停按钮上方**
3. 打开48V电源开关
4. 观察电机状态，确认无异常

### 步骤4：使能电机
```bash
# 使用脚本使能所有电机
python3 src/motor_tests_openarmx_com/en_all_motors.py
```

**注意事项：**
- ⚠️ 使能瞬间电机可能有**轻微抖动**，属正常现象
- ⚠️ 如发现**剧烈抖动或异响**，立即按下急停
- ⚠️ 使能后电机**保持当前位置**，小心触碰

### 步骤5：检查电机状态
```bash
# 检查所有电机状态
python3 src/motor_tests_openarmx_com/check_motor_status.py
```

**正常状态：**
- 所有电机响应正常
- 温度在合理范围（< 50°C）
- 无错误代码

### 步骤6：回零（首次启动或零位丢失时）
```bash
# 自动回零
python3 src/motor_tests_openarmx_com/control_motor_gohome.py
```

**⚠️ 回零注意事项：**
- 确保机械臂**初始位置接近零位**（±30度以内）
- 回零过程中**保持监控**
- 发现异常立即**按下急停**


##  日常操作规范

### 单电机测试
**使用场景：** 调试单个电机参数

**操作流程：**
```bash
#  启动单电机测试工具
python3 src/motor_tests_openarmx_com/test_motor_one.py

#  按提示输入参数
CAN通道: 0 或 1
电机ID: 1-8
位置: -12.5 ~ 12.5 (rad)
KP: 建议从 5.0 开始
KD: 建议从 0.5 开始
```

**安全要求：**
- ✅ 首次测试使用**小幅度运动**（±0.5 rad）
- ✅ 使用**保守的KP/KD参数**
- ✅ **持续监控**电机行为
- ❌ 不得超过**电机角度限制**

### MoveIt 运动规划
**使用场景：** 通过可视化界面控制机械臂

**操作流程：**
```bash
#  启动 MoveIt demo
ros2 launch openarm_moveit_config demo.launch.py

#  在 RViz 中操作
# - 选择 Planning Group（left_arm/right_arm）
# - 拖动交互标记到目标位置
# - 点击 "Plan" 生成轨迹
# - 检查轨迹安全性
# - 点击 "Execute" 执行
```

**安全要求：**
- ✅ 首次执行前**降低速度系数**（0.1-0.3）
- ✅ **仔细检查轨迹**，确认无碰撞风险
- ✅ 执行时**持续监控**
- ✅ 准备随时**按下急停**
- ❌ 不得在有障碍物时执行

### 轨迹录制
**使用场景：** 记录并回放运动轨迹

**操作流程：**
```bash
# 连续录制
ros2 run openarm_record record_joint_states_always

# 回放轨迹
ros2 run openarm_record play_joint_trajectory --file trajectory.yaml
```

**安全要求：**
- ✅ 录制前确保**轨迹路径安全**
- ✅ 回放前**预览轨迹文件**
- ✅ 首次回放使用**低速度**
- ❌ 不得回放未经验证的轨迹

### 示教模式
**使用场景：** 手动拖动机械臂录制轨迹

**操作流程：**
```bash
#  设置低 KP/KD 进入示教模式
# 关节1-4: KP=1.0, KD=0.1
# 关节5-7: KP=0.1, KD=0.05
# 夹爪: KP=0.1, KD=0.05

#  手动拖动机械臂
#  录制关键点
#  退出示教模式（恢复常规运动参数）
#    关节1-4: KP=50.0, KD=2.5
#    关节5-7: KP=10.0, KD=0.5
```

**安全要求：**
- ✅ 进入示教模式前确保**姿态安全**
- ✅ 拖动时**缓慢平稳**
- ⚠️ 注意退出示教模式时电机**会突然加力**


##  参数设置规范

### KP/KD参数设置

**关节1-4 (RS04/RS03电机):**

| 应用场景 | 推荐KP值 | 推荐KD值 | 说明 |
|---------|---------|---------|------|
| 示教模式 | 1.0 | 0.1 | 极低刚性，易于拖动 |
| 常规运动(默认) | 50.0 | 2.5 | 平衡精度和稳定性 |
| 高刚性模式 | 500.0 | 10.0 | 高精度，但可能产生噪声 |
| **危险范围** | > 5000 | > 100 | 可能导致振动、噪声、电机损坏 |

**关节5-7 (RS00电机):**

| 应用场景 | 推荐KP值 | 推荐KD值 | 说明 |
|---------|---------|---------|------|
| 示教模式 | 0.1 | 0.05 | 极低刚性，易于拖动 |
| 常规运动(默认) | 10.0 | 0.5 | 平衡精度和稳定性 |
| 高刚性模式 | 50.0 | 1.0 | 高精度，但可能产生噪声 |
| **危险范围** | > 500 | > 5 | 可能导致振动、噪声、电机损坏 |

**夹爪 (RS00电机):**

| 应用场景 | 推荐KP值 | 推荐KD值 |
|---------|---------|---------|
| 默认参数 | 5.0 | 0.5 |
| 示教模式 | 0.1 | 0.05 |
| 高刚性模式 | 50.0 | 1.0 |

**参数说明:**
- 以上参数来自代码：`openarmx_hardware/include/openarmx_hardware/v10_simple_hardware.hpp`
- **切勿超过危险范围**，否则可能损坏电机

### 速度和加速度限制
```yaml
# 推荐的速度/加速度限制
joint_limits:
  velocity: 2.0      # rad/s，初次测试建议 0.5
  acceleration: 1.0  # rad/s²，初次测试建议 0.3
```

### 参数调整原则
1. **从保守参数开始**
2. **逐步小幅增加**（每次增加10-20%）
3. **观察机器人行为**（声音、震动、稳定性）
4. **记录每次修改**和效果
5. **发现异常立即回退**


##  异常情况处理规范

### 立即急停的情况
遇到以下情况**立即按下急停按钮**：

- ❗ 机器人运动**超出预期范围**
- ❗ 电机发出**异常声音**（尖叫、噪声）
- ❗ 机械臂**剧烈震动**
- ❗ 电机或驱动器**冒烟、有焦味**
- ❗ 有人员**进入工作范围**
- ❗ CAN通信**频繁中断**
- ❗ 控制软件**失去响应**
- ❗ 任何**可能导致危险**的异常

### 急停后检查流程
```
1. 保持急停状态
2. 记录异常现象
3. 检查硬件状态：
   □ 电气连接是否松动
   □ 机械结构是否损坏
   □ 电机是否过热
4. 检查软件日志：
   □ 查看错误信息
   □ 检查参数配置
5. 排除故障后才能解除急停
6. 解除急停后从头开始检查流程
```

### 常见异常处理

**电机无响应：**
```bash
#  检查CAN连接
ip link show can0

#  重启CAN接口
sudo ip link set down can0
sudo ip link set up can0

#  重启PCAN连接（左右手反向时可重启PCAN或拔插一下PCAN接口）
sudo rmmod peak_usb
sudo modprobe peak_usb

#  检查电机状态
python3 check_motor_status.py
```

**电机抖动或噪声：**
```
1. 降低 KP 值
2. 增加 KD 值
3. 降低运动速度
4. 检查机械连接是否松动
```

**轨迹执行失败：**
```
1. 检查控制器状态
2. 检查目标位置是否超限
3. 降低速度和加速度
4. 检查 CAN 通信质量
```


##  关机规范

### 标准关机流程
```bash
# 步骤1：停止所有运动
# 确保机械臂处于安全姿态

# 步骤2：禁用电机
python3 src/motor_tests_openarmx_com/dis_all_motors.py

# 步骤3：关闭 CAN 接口
python3 src/motor_tests_openarmx_com/dis_all_can.py

# 步骤4：断开电源
# 关闭 48V 电源开关

# 步骤5：关闭控制计算机
```

### 紧急关机
**遇到严重故障时：**
1. 按下**急停按钮**
2. 直接断开**电源总开关**
3. **不要**尝试恢复，等待专业人员检查

2. **跳过**标准开机流程
3. 在**没有急停按钮**的情况下运行
4. 在工作区域**有人员**时高速运行
5. **未经测试**直接执行高速轨迹

8. **绕过**安全保护措施
9. **疲劳**或**注意力不集中**时操作
