# 紧急处理

## 紧急情况应急处理指南

本文档提供 OpenArmX 机器人系统紧急情况的应急处理流程，所有操作人员必须熟记。


##  紧急停止

### 急停按钮位置与使用

**急停按钮要求：**
- 🔴 **红色蘑菇头按钮**
- 🔴 位置**易于触及**（手臂伸展范围内）
- 🔴 **机械自锁式**（按下后保持锁定状态）
- 🔴 能够**立即切断所有电机电源**

**使用方法：**
```
按下：用手掌快速压下红色蘑菇头
解除：逆时针旋转蘑菇头使其弹起
```

### 需要立即急停的情况

**⚠️ 遇到以下情况立即按下急停按钮：**

| 情况 | 现象 | 危险程度 |
|------|------|---------|
| **失控运动** | 机械臂运动超出预期，速度过快 | 🔴 极高 |
| **剧烈震动** | 电机或机械臂剧烈抖动 | 🔴 极高 |
| **异常声音** | 尖叫、卡顿、金属摩擦声 | 🔴 极高 |
| **冒烟起火** | 电机、驱动器或线缆冒烟 | 🔴 极高 |
| **人员接近** | 有人员进入工作范围 | 🔴 极高 |
| **碰撞风险** | 即将碰撞障碍物或人员 | 🟠 高 |
| **通信中断** | CAN通信频繁断开 | 🟠 高 |
| **电机过热** | 电机温度>80°C | 🟠 高 |
| **软件崩溃** | 控制软件失去响应 | 🟠 高 |
| **电源异常** | 电压波动、频繁断电 | 🟠 高 |

### 急停后立即行动

**按下急停后：**

1. **保持急停状态** ✋
   - 不要立即解除急停
   - 防止设备重启后再次出现危险

2. **切断电源** 🔌
   - 如果急停后仍有异常（冒烟、异味）
   - 立即断开 24V 电源总开关

3. **撤离人员** 🚶
   - 确保所有人员撤离到安全区域
   - 建立警戒范围

4. **评估现场** 👀
   - 观察是否有明显损坏
   - 检查是否有人员受伤
   - 确认火灾风险


##  火灾应急

### 预防措施
```
✅ 配备 CO₂ 或干粉灭火器（禁用水基灭火器）
✅ 保持工作区域整洁，无易燃物
✅ 定期检查电气连接
✅ 不得超负荷运行电机
```

### 火灾处理流程

**小型电气火灾（初期）：**
```
1. 立即按下急停按钮
2. 断开电源总开关
3. 使用 CO₂ 或干粉灭火器
4. ❌ 严禁使用水灭火
5. 疏散人员，拨打 119
```

**大型火灾：**
```
1. 立即疏散所有人员
2. 拨打 119 报警
3. 不要尝试灭火
4. 等待消防人员到达
```


##  触电应急

### 触电预防
- 维护时必须**断开电源**
- 不得触碰**裸露的电气端子**
- 使用**绝缘工具**
- 穿**绝缘鞋**

### 触电救援

**发现有人触电：**

**⚠️ 错误做法：**
- ❌ 直接用手拉触电者
- ❌ 用湿手或导电物体接触

**✅ 正确做法：**
1. **立即切断电源** 🔌
   - 按下急停按钮
   - 断开电源总开关

2. **使用绝缘物体** 🪵
   - 使用干燥的木棍、塑料凳等
   - 将触电者与电源分离

3. **急救措施** 🏥
   - 检查意识和呼吸
   - 如无呼吸，立即进行心肺复苏（CPR）
   - 拨打 120 急救电话

4. **伤口处理** 🩹
   - 不要移动伤者（除非有二次危险）
   - 保持伤者温暖
   - 等待医护人员


##  机械伤害应急

### 夹伤/挤压伤

**发生夹伤时：**
```
1. 立即按下急停
2. 不要强行拉拽被夹部位
3. 断开电源
4. 手动移开机械臂
5. 检查伤情
6. 严重情况拨打 120
```

**轻度夹伤：**
- 用碘伏消毒
- 无菌敷料包扎
- 观察24小时

**严重夹伤（骨折、出血）：**
- 不要移动伤者
- 止血（压迫止血法）
- 固定伤处
- 立即拨打 120

### 碰撞伤

**头部碰撞：**
```
⚠️ 极度危险，即使看似轻微也要就医
1. 让伤者平躺
2. 不要移动头部
3. 观察意识状态
4. 立即拨打 120
```

**四肢碰撞：**
```
1. 检查是否骨折（疼痛、肿胀、变形）
2. 如有骨折，固定患处
3. 冷敷减轻肿胀
4. 必要时就医
```


##  设备故障应急

### 电机失控

**现象：** 电机高速旋转、无法停止

**处理流程：**
```
1. 立即按下急停 ⚠️
2. 断开电源
3. 等待电机完全停止（至少30秒）
4. 检查CAN连接
5. 检查控制软件日志
6. 重新校准零位
7. 低速测试
```

### CAN通信故障

**现象：** 频繁报错、电机无响应

**处理流程：**
```bash
#  检查CAN接口状态
ip link show can0
ip link show can1

#  检查错误计数
ip -details -statistics link show can0

#  重启CAN接口
sudo ip link set down can0
sudo ip link set up can0

#  检查物理连接
# - 检查线缆是否松动
# - 检查是否有断路
# - 检查终端电阻（120Ω）

#  降低波特率测试
sudo ip link set can0 type can bitrate 500000
```

### 电机过热

**现象：** 电机温度 > 80°C

**⚠️ 危险等级：**
- 60-70°C：🟡 正常（高负载）
- 70-80°C：🟠 警告（需降低负载）
- 80-90°C：🔴 危险（立即停止）
- \>90°C：🔴🔴 极度危险（可能损坏）

**处理流程：**
```
1. 立即停止运动
2. 禁用电机
3. 自然冷却（不要用水冷却）
4. 检查原因：
   □ 负载是否过大
   □ KP/KD 参数是否过高
   □ 运动速度是否过快
   □ 散热是否良好
5. 冷却后重新测试
```

### 零位丢失

**现象：** 机械臂位置与软件显示不一致

**⚠️ 危险：** 可能导致突然运动

**处理流程：**
```
1. 立即禁用电机
2. 断电
3. 手动将机械臂移至安全姿态
4. 重新上电
5. 重新设置零位
6. 低速验证
```


##  软件故障应急

### 控制软件崩溃

**现象：** ROS节点崩溃、无响应

**处理流程：**
```bash
#  立即按下急停或禁用电机
python3 src/motor_tests_openarmx_com/dis_all_motors.py

#  检查进程状态
ps aux | grep ros

#  强制终止所有ROS进程
killall -9 ros2

#  检查日志
ros2 log

#  重启系统（如需要）
```

### 参数配置错误

**现象：** 机械臂行为异常

**处理流程：**
```yaml
#  立即停止运动
#  恢复默认参数
joint_limits:
  velocity: 0.5
  acceleration: 0.3

kp_default: [10, 10, 10, 10, 5, 5, 5, 5]
kd_default: [0.5, 0.5, 0.5, 0.5, 0.5, 0.5, 0.5, 0.5]

#  重新启动系统
#  逐步调整参数
```

##  事故后处理

### 事故调查
```
1. 保护现场
2. 记录现场情况（拍照）
3. 收集证据和日志
4. 访谈相关人员
5. 分析事故原因
6. 提出改进措施
```

### 设备恢复
```
⚠️ 不得在未经检查的情况下重启设备

检查清单：
□ 电气系统完好
□ 机械结构无损
□ 安全装置功能正常
□ 零位校准正确
□ 参数配置合理
□ 进行低速测试
```

### 预防措施
```
1. 更新操作规程
2. 加强人员培训
3. 改进安全设施
4. 定期检查维护
5. 记录经验教训
```
