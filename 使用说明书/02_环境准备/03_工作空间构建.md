# 03_工作空间构建

## OpenArmX ROS2工作空间构建指南

本文档详细说明如何克隆OpenArmX代码,配置工作空间并编译所有软件包。

---

## 1. 工作空间概念

### 1.1 ROS2工作空间结构

ROS2工作空间是一个包含源代码、编译输出和安装文件的目录结构:

```
~/openarmx_robotstride_ws-cc/     # 工作空间根目录
├── src/                          # 源代码目录
│   ├── openarm_ros2/             # 功能包1
│   ├── openarm_moveit_config/    # 功能包2
│   └── ...                       # 其他功能包
│
├── build/                        # 编译输出(colcon自动生成)
│   ├── openarm_ros2/
│   └── ...
│
├── install/                      # 安装目录(colcon自动生成)
│   ├── setup.bash                # 环境设置脚本
│   ├── openarm_ros2/
│   └── ...
│
└── log/                          # 编译日志(colcon自动生成)
    └── build_YYYY-MM-DD_HH-MM-SS/
```

### 1.2 工作空间流程

```
下载源码 → 安装依赖 → 编译 → 加载环境 → 运行
  (git)    (rosdep)   (colcon)  (source)
```

## 2. 创建工作空间

### 2.1 创建工作空间目录

```bash
# 进入家目录
cd ~

# 创建工作空间
mkdir -p ~/openarmx_robotstride_ws-cc/src

# 进入src目录
cd ~/openarmx_robotstride_ws-cc/src
```

**⚠️ 注意:**
- 工作空间名称可以自定义,但本文档统一使用`openarmx_robotstride_ws-cc`
- `src`目录必须存在,colcon才能识别为工作空间

## 3. 克隆源代码

### 3.1 获取OpenArmX代码

**方法1: 从Git仓库克隆(推荐):**

```bash
cd ~/openarmx_robotstride_ws-cc/src

# 克隆主仓库(请替换为实际URL)
git clone https://github.com/openarmx/openarmx_robotstride_ws.git .

# 或使用Gitee镜像(国内用户)
git clone https://gitee.com/openarmx/openarmx_robotstride_ws.git .

ls -l
```

**方法2: 从压缩包解压:**

```bash
cd ~/openarmx_robotstride_ws-cc/src

# 假设已下载openarmx_src.tar.gz
tar -xzf ~/Downloads/openarmx_src.tar.gz -C .

# 查看解压内容
ls -l
```

### 3.2 验证源代码结构

```bash
# 检查主要功能包
cd ~/openarmx_robotstride_ws-cc/src
ls -d */

# 应该看到:
# openarm_ros2/
# openarm_moveit_config/
# openarm_record/
# openarmx_can/
# motor_tests_openarmx_com/
# motor_manage_openarmx/
# 使用说明书/
```

### 3.3 检查package.xml文件

```bash
# 查找所有ROS2功能包
find ~/openarmx_robotstride_ws-cc/src -name "package.xml"

# 每个ROS2功能包应该包含package.xml
```

## 4. 安装依赖

### 4.1 使用rosdep安装系统依赖

```bash
# 回到工作空间根目录
cd ~/openarmx_robotstride_ws-cc

# 使用rosdep安装依赖
rosdep install --from-paths src --ignore-src -r -y
```

**命令解释:**
- `--from-paths src`: 扫描src目录下所有package.xml
- `--ignore-src`: 忽略已在src中的包
- `-r`: 继续安装,即使某些包失败
- `-y`: 自动确认安装

**预期输出:**
```
#All required rosdeps installed successfully
```

**⚠️ 如果遇到错误:**
```bash
# 更新rosdep数据库
rosdep update

# 重试安装
rosdep install --from-paths src --ignore-src -r -y
```

### 4.2 安装Python依赖

某些Python依赖可能不在rosdep数据库中,需手动安装:

```bash
# 进入工作空间
cd ~/openarmx_robotstride_ws-cc

# 安装Python CAN库
pip3 install python-can

# 安装GUI库(电机管理界面)
pip3 install PySide6

# 其他可能需要的库
pip3 install numpy scipy matplotlib pyyaml
```

## 5. 编译工作空间

### 5.1 首次编译

```bash
# 回到工作空间根目录
cd ~/openarmx_robotstride_ws-cc

# 编译所有包
colcon build

# 编译过程可能需要5-15分钟,取决于硬件性能
```

**编译选项:**

```bash
# 并行编译(加速)
colcon build --parallel-workers 4

# 带详细输出
colcon build --event-handlers console_direct+

# 仅编译特定包
colcon build --packages-select openarm_ros2

# 编译某个包及其依赖
colcon build --packages-up-to openarm_moveit_config
```

### 5.2 预期编译输出

**成功输出:**
```
Starting >>> openarmx_description
Starting >>> openarmx_can
...
Finished <<< openarmx_description [12.3s]
Finished <<< openarmx_can [45.6s]
...
Summary: 8 packages finished [1min 23s]
```

**文件变化:**
```
工作空间目录下新增:
├── build/      # 编译中间文件
├── install/    # 安装文件
└── log/        # 编译日志
```

### 5.3 编译错误处理

**常见编译错误:**

**错误1: 缺少依赖**
```
CMake Error: Could not find package "xxx"
```
**解决:**
```bash
# 安装缺失的ROS2包
sudo apt install ros-humble-xxx

# 或使用rosdep
rosdep install --from-paths src --ignore-src -r -y
```

**错误2: Python模块缺失**
```
ModuleNotFoundError: No module named 'xxx'
```
**解决:**
```bash
pip3 install xxx
```

**错误3: 编译器版本问题**
```
error: 'xxx' was not declared in this scope
```
**解决:**
```bash
# 确保使用GCC 11+
gcc --version

# 如果版本过低,升级编译器
sudo apt install gcc-11 g++-11
```

## 6. 配置环境变量

### 6.1 加载工作空间环境

**临时加载(当前终端):**
```bash
cd ~/openarmx_robotstride_ws-cc
source install/setup.bash
```

**永久加载(推荐):**
```bash
# 编辑.bashrc
vim ~/.bashrc

# 在文件末尾添加:
source /opt/ros/humble/setup.bash
source ~/openarmx_robotstride_ws-cc/install/setup.bash

# 保存并退出

# 重新加载.bashrc
source ~/.bashrc
```

**⚠️ 重要提示:**
- `source /opt/ros/humble/setup.bash` 必须在工作空间之前
- 每次编译后需要重新source

### 6.2 环境变量说明

```bash
# 查看ROS2环境变量
printenv | grep ROS

# 关键变量:
# ROS_DISTRO=humble
# ROS_VERSION=2
# AMENT_PREFIX_PATH=.../install/...
# CMAKE_PREFIX_PATH=.../install/...
```

### 6.3 验证环境配置

```bash
# 查看ROS2包路径
ros2 pkg list | grep openarm

# 应该看到:
# openarmx_bringup
# openarm_controllers
# openarmx_description
# openarmx_hardware
# openarm_moveit_config
# ...

# 检查包路径
ros2 pkg prefix openarmx_description
# 输出: /home/xxx/openarmx_robotstride_ws-cc/install/openarmx_description
```

## 8. 工作空间管理

### 8.1 清理编译文件

**清理build和install:**
```bash
cd ~/openarmx_robotstride_ws-cc

# 删除编译输出
rm -rf build/ install/ log/

# 重新编译
colcon build
```

**仅清理特定包:**
```bash
# 删除特定包的编译文件
rm -rf build/openarm_ros2 install/openarm_ros2

# 重新编译该包
colcon build --packages-select openarm_ros2
```

### 8.2 增量编译

修改代码后,colcon会自动增量编译:

```bash
# 修改了某个文件后
cd ~/openarmx_robotstride_ws-cc
colcon build

# colcon会检测变化,只编译修改的包
```

**强制重新编译:**
```bash
# 强制全部重新编译
colcon build --cmake-clean-cache

# 强制特定包重新编译
colcon build --packages-select openarm_ros2 --cmake-clean-cache
```

### 8.3 并行编译优化

```bash
# 使用所有CPU核心
colcon build --parallel-workers $(nproc)

# 限制并发数(避免内存不足)
colcon build --parallel-workers 2

# 设置为默认(添加到.bashrc)
echo "export COLCON_PARALLEL_WORKERS=4" >> ~/.bashrc
```

## 10. 多工作空间管理

### 10.1 Overlay概念

ROS2支持工作空间叠加(Overlay):

```
系统ROS2 (/opt/ros/humble)
    ↓ underlay
工作空间1 (~/openarmx_robotstride_ws-cc)
    ↓ underlay
工作空间2 (~/my_custom_ws)  ← 最高优先级
```

### 10.2 创建Overlay工作空间

```bash
# 创建新工作空间
mkdir -p ~/my_custom_ws/src
cd ~/my_custom_ws/src

# 添加自定义包
git clone <your_custom_package>

# 回到新工作空间根目录
cd ~/my_custom_ws

# 先source基础工作空间
source ~/openarmx_robotstride_ws-cc/install/setup.bash

# 编译新工作空间
colcon build

# 加载新工作空间
source install/setup.bash

# 现在新工作空间的包会覆盖基础工作空间的同名包
```

## 11. 常用colcon命令

### 11.1 编译命令

```bash
# 基本编译
colcon build

# 详细输出
colcon build --event-handlers console_direct+

# 仅编译一个包
colcon build --packages-select <package_name>

# 编译包及其依赖
colcon build --packages-up-to <package_name>

# 跳过某些包
colcon build --packages-skip <package_name>

# 从某个包开始编译
colcon build --packages-start <package_name>

# 并行编译
colcon build --parallel-workers 4

# CMake编译类型(Debug/Release)
colcon build --cmake-args -DCMAKE_BUILD_TYPE=Release
```

### 11.2 测试命令

```bash
# 运行测试
colcon test

# 查看测试结果
colcon test-result --all

# 详细测试输出
colcon test-result --verbose
```

### 11.3 信息查询

```bash
# 列出所有包
colcon list

# 列出包的依赖关系
colcon list --packages-select openarm_ros2 --base-paths src

# 列出包的拓扑顺序
colcon list --topological-order
```

## 13. 故障排查

### 13.1 colcon build失败

**症状:** 编译过程中断

**排查步骤:**
```bash
# 1. 查看详细日志
cd ~/openarmx_robotstride_ws-cc
colcon build --event-handlers console_direct+

# 2. 查看特定包的日志
cat log/latest_build/openarm_ros2/stdout.log
cat log/latest_build/openarm_ros2/stderr.log

# 3. 检查依赖
rosdep check --from-paths src --ignore-src

# 4. 清理后重试
rm -rf build/ install/ log/
colcon build
```

### 13.2 找不到包

**症状:** `Package 'openarm_xxx' not found`

**解决:**
```bash
# 1. 确认已source环境
source ~/openarmx_robotstride_ws-cc/install/setup.bash

# 2. 检查包是否存在
ros2 pkg list | grep openarm

# 3. 重新编译
cd ~/openarmx_robotstride_ws-cc
colcon build --packages-select openarm_xxx
source install/setup.bash
```

### 13.3 环境变量冲突

**症状:** 同时安装了多个ROS版本

**解决:**
```bash
# 检查当前ROS环境
printenv | grep ROS

# 清除所有ROS环境变量
unset ROS_DISTRO
unset AMENT_PREFIX_PATH
unset CMAKE_PREFIX_PATH

# 重新source正确的环境
source /opt/ros/humble/setup.bash
source ~/openarmx_robotstride_ws-cc/install/setup.bash
```

## 14. 开发工作流

### 14.1 典型开发流程

```bash
# 1. 修改代码
vim ~/openarmx_robotstride_ws-cc/src/openarm_ros2/...

# 使用
~/rebuild.sh
```

## 15. 性能优化

### 15.1 编译加速

**使用ccache:**
```bash
# 安装ccache
sudo apt install -y ccache

# 配置colcon使用ccache
export CC="ccache gcc"
export CXX="ccache g++"

# 编译
colcon build
```

**使用RAM磁盘(适合大内存):**
```bash
# 创建RAM磁盘
sudo mkdir -p /mnt/ramdisk
sudo mount -t tmpfs -o size=4G tmpfs /mnt/ramdisk

# 在RAM磁盘中编译
cd ~/openarmx_robotstride_ws-cc
colcon build --build-base /mnt/ramdisk/build
```

### 15.2 symlink-install

Python包使用符号链接安装,修改后无需重新编译:

```bash
colcon build --symlink-install

# 之后修改Python代码,无需重新编译,直接运行即可
```

**⚠️ 注意:** C++代码仍需重新编译
