# 01_CAN总线连接

## CAN总线物理连接指南

本文档详细说明OpenArmX系统的CAN总线物理连接方法和验证步骤。

---

## 1. CAN总线概述

### 1.1 为什么使用双CAN总线

OpenArmX采用**双CAN总线架构**:

```
can0: 左臂(7个关节) + 左夹爪(1个) = 8个电机
can1: 右臂(7个关节) + 右夹爪(1个) = 8个电机
```

**优势:**
- ✅ **降低总线负载**: 每条总线8个设备,通信更流畅
- ✅ **故障隔离**: 一侧故障不影响另一侧
- ✅ **提高可靠性**: 减少CAN帧冲突
- ✅ **独立控制**: 左右臂可独立操作

### 1.2 CAN总线基本原理

**CAN 2.0 特性:**
- 差分信号传输(CAN_H, CAN_L)
- 最高波特率: 1 Mbps
- 多主机通信
- 自动仲裁和错误检测

**信号线:**
```
CAN_H: 高电平线(显性3.5V, 隐性2.5V)
CAN_L: 低电平线(显性1.5V, 隐性2.5V)
GND:   信号地线
```

## 2. 硬件准备

### 2.1 所需设备清单

```
必需设备:
□ OpenArmX机器人本体(含16个电机)
□ 双通道CAN-USB转换器 × 1
  (或单通道CAN-USB转换器 × 2)
□ CAN线缆(双绞屏蔽线)
```

```
注意:
- 单通道,需要2个
```

## 3. CAN线缆连接

### 3.1 DB9接口针脚定义

**标准CAN DB9针脚(CiA DS-102):**

```
DB9母头(从前面看):
  ┌─────────┐
  │ 5 4 3 2 1 │
  │  9 8 7 6  │
  └─────────┘

针脚定义:
Pin 1: (保留)
Pin 2: CAN_L      ← 主要使用
Pin 3: GND        ← 主要使用
Pin 4: (保留)
Pin 5: CAN_SHLD (屏蔽层,可选)
Pin 6: GND (可选)
Pin 7: CAN_H      ← 主要使用
Pin 8: (保留)
Pin 9: CAN_V+ (可选,外部供电)
```

**⚠️ 重要:**
不同厂商可能有不同定义,**连接前务必查阅设备手册**!

### 3.2 电机端接线

**Robstride电机CAN接口:**

OpenArmX的Robstride电机通常使用以下接线方式:

```
电机端子(典型):
┌──────────┐
│ CAN_H    │ ← 黄线/橙线
│ CAN_L    │ ← 蓝线/白线
│ GND      │ ← 黑线
│ 48V+     │ ← 红线(电源)
└──────────┘
```

**⚠️ 注意:**
- 电机CAN接口可能是端子、航插或DB9
- 具体以实际产品为准
- 电源线(48V+, GND)与CAN信号线分开

### 3.3 完整连接拓扑

**左臂(can0)连接示意:**

```
控制电脑                CAN转换器              左臂电机链
┌────────┐            ┌──────────┐
│  USB   │            │  DB9     │  CAN_H ───┬─ 电机1 (ID=1)
│  接口  ├───USB线────►│  can0    │           │
└────────┘            │          │  CAN_L ───┼─ 电机2 (ID=2)
                      └──────────┘           │
                           │                 ├─ 电机3 (ID=3)
                      120Ω终端               │
                                             ├─ 电机4 (ID=4)
                                             │
                                             ├─ 电机5 (ID=5)
                                             │
                                             ├─ 电机6 (ID=6)
                                             │
                                             ├─ 电机7 (ID=7)
                                             │
                                             └─ 电机8 (ID=8)
                                                  │
                                             120Ω终端
```

**右臂(can1)连接方式相同**

### 3.4 菊花链vs星型拓扑

**推荐: 菊花链(Daisy Chain)**

```
转换器 ─┬─ 电机1 ─┬─ 电机2 ─┬─ 电机3 ─┬─ ... ─ 电机8
        │         │         │         │
      120Ω      (无)       (无)      (无)      120Ω
```

**优点:**
- ✅ 布线简单
- ✅ 电缆用量少
- ✅ 符合CAN标准

**❌ 不推荐: 星型拓扑**

```
        ┌─ 电机1
        ├─ 电机2
转换器 ─┼─ 电机3
        ├─ 电机4
        └─ ...
```

**缺点:**
- 反射严重
- 可靠性差
- 不符合CAN规范

## 4. 终端电阻配置

### 4.1 为什么需要终端电阻

CAN总线是传输线,需要在**两端**安装120Ω电阻消除信号反射。

```
起点           总线            终点
120Ω ─────────────────────── 120Ω
      CAN_H
      CAN_L
```

### 4.2 终端电阻位置

**典型配置:**

```
can0总线:
- 位置1: CAN转换器内部(通常已集成)
- 位置2: 最后一个电机(电机8)或外接电阻

can1总线:
- 位置1: CAN转换器内部
- 位置2: 最后一个电机(电机8)或外接电阻
```

### 4.3 检查终端电阻

**方法1: 万用表测量**

```bash
# 1. 断开所有设备电源
# 2. 将万用表设置为电阻档
# 3. 测量CAN_H和CAN_L之间电阻

预期结果:
- 两个120Ω电阻并联 ≈ 60Ω
- 如果测量值在55-65Ω,说明正常
- 如果>100Ω,说明缺少终端电阻
- 如果<50Ω,说明电阻过多或短路
```

**方法2: 查看设备手册**

检查CAN转换器和电机是否内置终端电阻,以及如何启用/禁用。

### 4.4 外接终端电阻

如果设备没有内置终端电阻:

```
材料:
□ 120Ω电阻(1/4W或更高功率)
□ DB9连接器或端子

接法:
CAN_H ──┬── 120Ω ──┬── CAN_L
        │          │
      (Pin 7)    (Pin 2)
```

## 5. 实际接线步骤

### 5.1 准备工作

```
1. 确认所有设备已断电
2. 准备好所有线缆和工具
3. 查阅设备接口定义
4. 规划线缆走向
```

### 5.2 连接CAN转换器到电脑

```bash
# 1. 将CAN-USB转换器通过USB线连接到电脑

# 2. 检查是否识别
lsusb

# 应该看到类似:
# Bus 001 Device 005: ID 0c72:000c ... PEAK PCAN-USB
# 或
# Bus 001 Device 006: ID ... ... USBCAN-II

# 3. 检查内核日志
dmesg | tail -20

# 应该看到CAN设备加载信息
```

### 5.3 连接CAN线缆到电机

**步骤:**

```
1. 从CAN转换器引出CAN线缆
   - CAN_H连接到所有电机的CAN_H
   - CAN_L连接到所有电机的CAN_L
   - GND连接到GND(或共用电源地)

2. 按照菊花链方式连接各电机
   转换器 → 电机1 → 电机2 → ... → 电机8

3. 确保接线牢固
   - 螺丝端子拧紧
   - 航插完全插入并锁定
   - DB9连接器固定螺丝拧紧

4. 检查接线
   - CAN_H到CAN_H
   - CAN_L到CAN_L
   - 没有接反
   - 没有短路
```

### 5.4 电源连接

**⚠️ 注意: 电源连接单独处理,不与CAN信号线混淆**

```
48V电源 ──┬── 电机1 (48V+, GND)
          ├── 电机2
          ├── 电机3
          ├── ...
          └── 电机8

推荐使用:
- 足够粗的电源线(≥16AWG)
- 带保险丝的配电板
- 急停继电器
```

## 6. 接线验证

### 6.1 视觉检查

```
□ 所有连接器插紧
□ 没有裸露的金属线
□ 线缆走向整齐,无交叉缠绕
□ 没有线缆拉扯过紧
□ 屏蔽层正确接地(如使用屏蔽线)
□ 终端电阻已正确安装
```

### 6.2 电阻测量

```bash
# 断电状态下测量CAN_H和CAN_L之间电阻
# 使用万用表

can0: CAN_H <-> CAN_L ≈ 60Ω
can1: CAN_H <-> CAN_L ≈ 60Ω

如果阻值不对:
- >100Ω: 检查终端电阻是否安装
- <50Ω: 检查是否短路或电阻过多
- ∞(开路): 检查线缆连接
```

### 6.3 上电测试

**⚠️ 首次上电前确认:**
```
□ CAN接线正确
□ 电源接线正确(极性)
□ 终端电阻已安装
□ 急停按钮可用
□ 人员已撤离
```

**上电步骤:**
```
1. 连接USB到电脑
2. 启动CAN接口(见下节)
3. 打开48V电源
4. 观察:
   - 电源指示灯亮起
   - 无异常声音
   - 无冒烟
   - CAN转换器指示灯正常
```

## 7. CAN接口配置

### 7.1 加载CAN驱动

```bash
# 加载SocketCAN内核模块
sudo modprobe can
sudo modprobe can_raw

# 验证模块加载
lsmod | grep can
```

### 7.2 配置can0和can1

**使用脚本(推荐):**

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

# 启动CAN接口
python3 en_all_can.py
```

**手动配置:**

```bash
# 配置can0
sudo ip link set can0 type can bitrate 1000000
sudo ip link set can0 up

# 配置can1
sudo ip link set can1 type can bitrate 1000000
sudo ip link set can1 up

# 检查状态
ip link show can0
ip link show can1

# 应该看到:
# can0: <NOARP,UP,LOWER_UP,ECHO> ...
# can1: <NOARP,UP,LOWER_UP,ECHO> ...
```

**⚠️ PCAN设备特殊处理:**

如果遇到左右臂反向,可以重启PCAN模块:

```bash
# 卸载PCAN模块
sudo rmmod peak_usb

# 重新加载
sudo modprobe peak_usb

# 或者拔插USB接口

# 重新配置CAN
python3 en_all_can.py
```

### 7.3 验证CAN接口

```bash
# 检查CAN接口状态
ip -details link show can0
ip -details link show can1

# 关键信息:
# - state UP (接口已启动)
# - bitrate 1000000 (波特率正确)
# - RX/TX错误计数器为0或很小
```

## 8. 通信测试

### 8.1 监听CAN总线

**使用candump:**

```bash
# 监听can0
candump can0

# 监听can1
candump can1

# 同时监听
candump any

# 带时间戳
candump -t a can0
```

**预期现象:**
- 如果电机已上电,应该看到CAN帧
- 帧ID范围: 0x00 ~ 0xFF
- 持续有数据流动

### 8.2 发送测试帧

**⚠️ 注意: 仅用于测试,不要随意发送命令给电机**

```bash
# 发送测试帧到can0 (不会影响电机)
cansend can0 123#DEADBEEF

# 在另一终端监听
candump can0

# 应该能看到刚发送的帧
```

### 8.3 检查电机响应

**使用电机状态检查脚本:**

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

# 检查所有电机状态
python3 check_motor_status.py
```

**预期输出:**
```
CAN0 电机状态:
ID 1: RS04 - 位置: 0.123 rad, 温度: 45°C
ID 2: RS04 - 位置: -0.056 rad, 温度: 43°C
...
ID 8: RS00 - 位置: 0.000 rad, 温度: 38°C

CAN1 电机状态:
ID 1: RS04 - 位置: 0.089 rad, 温度: 44°C
...
```

**如果某些电机无响应:**
- 检查该电机的CAN连接
- 检查电机ID配置
- 检查电机供电

## 9. 常见问题排查

### 9.1 CAN接口无法启动

**症状:**
```bash
$ sudo ip link set can0 up
Error: Cannot find device "can0"
```

**解决:**
```bash
# 1. 检查USB设备
lsusb

# 2. 检查内核日志
dmesg | grep -i can

# 3. 重新插拔USB
# 4. 检查驱动是否加载
lsmod | grep can
```

### 9.2 CAN总线错误率高

**症状:**
```bash
$ ip -s link show can0
RX: errors 150  # 接收错误多
TX: errors 80   # 发送错误多
```

**可能原因和解决:**

**1. 终端电阻问题**
```bash
# 检查阻值应为60Ω
# 添加或移除电阻
```

**2. 波特率不匹配**
```bash
# 确认所有设备使用1Mbps
sudo ip link set can0 type can bitrate 1000000
```

**3. 线缆质量差**
```
- 更换屏蔽双绞线
- 减短线缆长度(<5m)
- 远离强电磁干扰源
```

**4. 信号质量差**
```bash
# 降低波特率测试
sudo ip link set can0 down
sudo ip link set can0 type can bitrate 500000
sudo ip link set can0 up
```

### 9.3 部分电机无响应

**排查步骤:**

```bash
# 1. 使用candump监听总线
candump can0

# 2. 检查哪些ID有响应
# 3. 无响应电机:
#    - 检查该电机的CAN接线
#    - 检查电机供电
#    - 检查电机ID设置

# 4. 尝试单独连接该电机
#    - 移除其他电机,只接一个
#    - 如果能通信,说明电机正常
#    - 逐个添加其他电机,找出问题所在
```

### 9.4 CAN总线风暴(Bus-Off)

**症状:**
```bash
$ ip link show can0
can0: <NOARP,UP,LOWER_UP,ECHO> ... state ERROR-PASSIVE
# 或
can0: ... state BUS-OFF
```

**原因:**
- 错误计数器超过阈值
- 可能存在短路或通信冲突

**解决:**
```bash
# 1. 重启CAN接口
sudo ip link set can0 down
sudo ip link set can0 up

# 2. 检查总线是否有持续错误帧
candump can0

# 3. 检查物理连接
#    - 有无短路
#    - 终端电阻是否正确

# 4. 如果频繁Bus-Off
#    - 降低波特率
#    - 检查所有设备是否正常
```

## 10. 安全注意事项

### 10.1 接线安全

```
⚠️ 断电操作
- 接线时务必断开48V电源
- 避免带电插拔CAN连接器

⚠️ 防静电
- 触摸电机前先放电(触摸接地金属)
- 避免静电损坏电机控制器

⚠️ 绝缘检查
- 确保裸露导体无短路风险
- 使用绝缘胶带包裹临时接头
```

### 10.2 调试安全

```
⚠️ 不要发送未知命令
- 不了解CAN协议时不要乱发命令
- 可能导致电机失控

⚠️ 急停准备
- 调试时手放在急停按钮上
- 发现异常立即按下急停

⚠️ 逐步测试
- 先测试通信,再测试运动
- 从小幅度、低速度开始
```
