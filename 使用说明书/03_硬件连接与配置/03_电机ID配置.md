# 03_电机ID配置

## 电机ID设置和零位校准指南

本文档说明如何查看、设置电机ID,以及如何校准零位。

---

## 1. 电机ID概述

### 1.1 ID分配方案

OpenArmX的16个电机按以下规则分配ID:

**can0 (左臂+左夹爪):**
```
ID 1: 左肩关节1 (RS04) - 肩部
ID 2: 左肩关节2 (RS04) - 肩部
ID 3: 左大臂关节3 (RS03) - 大臂
ID 4: 左大臂关节4 (RS03) - 大臂
ID 5: 左小臂关节5 (RS00) - 小臂
ID 6: 左小臂关节6 (RS00) - 小臂
ID 7: 左小臂关节7 (RS00) - 小臂
ID 8: 左夹爪 (RS00) - 夹爪
```

**can1 (右臂+右夹爪):**
```
ID 1: 右肩关节1 (RS04) - 肩部
ID 2: 右肩关节2 (RS04) - 肩部
ID 3: 右大臂关节3 (RS03) - 大臂
ID 4: 右大臂关节4 (RS03) - 大臂
ID 5: 右小臂关节5 (RS00) - 小臂
ID 6: 右小臂关节6 (RS00) - 小臂
ID 7: 右小臂关节7 (RS00) - 小臂
ID 8: 右夹爪 (RS00) - 夹爪
```

### 1.2 ID的重要性

- ✅ 每条总线上ID必须唯一
- ✅ ID决定了控制命令的目标电机
- ❌ ID冲突会导致控制混乱
- ❌ ID错误会导致电机无响应

## 3. 零位概念

### 3.1 什么是零位

**零位**是机器人的**标准初始姿态**,所有关节角度定义为0。

**OpenArmX零位姿态:**
```
- 双臂垂直下垂
- 肩部、肘部、腕部伸直
- 夹爪完全张开
```

### 3.2 为什么要设置零位

- ✅ 统一关节角度参考点
- ✅ 保证运动学计算正确
- ✅ 避免启动时突然运动
- ✅ 便于轨迹规划

### 3.3 零位丢失的影响

如果零位不准确:
- ❌ 机器人姿态与软件显示不符
- ❌ 运动轨迹错误
- ❌ 可能碰撞或超限
- ❌ 末端位置偏差

## 4. 零位设置步骤

### 4.1 准备工作

```
前提条件:
□ CAN接口已配置并启动
□ 48V电源已连接
□ 急停按钮可用
□ 了解安全操作规范
```

### 4.2 调整到零位姿态

**手动调整机械臂:**

```
1. 断开48V电源(或禁用电机)

2. 手动将机械臂调整到零位姿态:
   - 肩部关节: 垂直向下
   - 肘部关节: 伸直
   - 腕部关节: 中立位置
   - 夹爪: 完全张开

3. 确保各关节在±30度范围内
   (超过30度设置零位脚本会报错)

4. 检查左右臂姿态对称
```

### 4.3 使用GUI设置零位

```bash
# 启动电机管理界面
cd ~/openarmx_robotstride_ws-cc/src/motor_manage_openarmx
python3 main.py

# 界面操作:
# 1. 点击"启动CAN"
# 2. 点击"启动电机"
# 3. 确认机械臂已在零位姿态
# 4. 点击"设置零位"
# 5. 等待设置完成
```

### 4.4 使用命令行脚本设置

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

# 1. 启动CAN接口
python3 en_all_can.py

# 2. 启动电机
python3 en_all_motors.py

# 3. 设置零位
python3 set_motor_zeros_setMIT.py
```

**脚本执行过程:**
```
正在读取电机位置...
can0 ID 1: 当前位置 0.05 rad (OK)
can0 ID 2: 当前位置 -0.03 rad (OK)
...
can1 ID 8: 当前位置 0.00 rad (OK)

所有电机位置偏差 < 0.52 rad (30度)
正在设置零位...
can0 ID 1: 零位设置成功
can0 ID 2: 零位设置成功
...
完成! 所有电机零位已设置
```

**⚠️ 如果出现错误:**
```
错误: can0 ID 1 位置偏差过大: 0.85 rad (> 0.52 rad)
请手动调整机械臂到零位姿态后重试
```

**解决方法:**
- 手动调整该关节到零位附近
- 重新运行脚本

### 4.5 零位设置原理

**MIT模式设置零位命令:**

Robstride电机使用特定CAN命令设置零位:

```
CAN ID: 电机ID + 特定偏移
数据: 零位设置命令帧
功能: 将当前位置设为0点
保存: 写入电机Flash存储
```

**重启后生效:**
设置零位后需要**重启电机**(断电重启或禁用再使能)

## 6. 常见问题

### 6.1 设置零位失败

**问题:** 脚本报错"位置偏差过大"

**原因:**
- 机械臂未调整到零位附近
- 某个关节卡住
- 电机未使能

**解决:**
```bash
# 1. 检查电机是否使能
python3 check_motor_status.py

# 2. 手动调整机械臂
#    - 确保各关节能自由活动
#    - 调整到零位姿态

# 3. 重新设置零位
python3 set_motor_zeros_setMIT.py
```

### 6.2 零位设置后仍然偏差大

**问题:** 设置零位后,回零姿态不正确

**可能原因:**
1. 设置零位时机械臂姿态不正确
2. 设置后未重启电机
3. 电机编码器故障

**解决:**
```bash
# 1. 断电重启电机
# 关闭48V电源,等待5秒,重新上电

# 2. 重新使能电机
python3 en_all_motors.py

# 3. 检查位置
python3 check_motor_status.py

# 4. 如果仍然偏差大,重新设置零位
```

### 6.3 部分电机零位正确,部分错误

**排查步骤:**

```bash
# 1. 检查哪些电机零位错误
python3 check_motor_status.py

# 2. 仅针对错误电机重新设置
#    修改set_motor_zeros_setMIT.py
#    或使用GUI单独设置

# 3. 检查错误电机的机械连接
#    - 是否松动
#    - 是否有卡顿
```

## 8. 高级操作

### 8.1 单独设置某个电机零位

**修改脚本:**

```python
# 示例: 仅设置can0 ID 7的零位
import can
from lib.motor_control import set_zero_position

bus = can.interface.Bus(channel='can0', bustype='socketcan')
motor_id = 7

# 设置零位
set_zero_position(bus, motor_id)
print(f"can0 ID {motor_id} 零位已设置")
```

### 8.2 批量设置零位

如果有多个机器人:

```bash
# 为每个机器人创建零位配置文件
# robot1_zeros.yaml
# robot2_zeros.yaml

# 使用配置文件设置零位
python3 set_zeros_from_config.py --config robot1_zeros.yaml
```

### 8.3 零位偏移补偿

如果机械安装有固定偏差:

```python
# 在URDF或控制器中添加偏移
joint_offset = {
    'left_joint_1': 0.05,  # 5度偏移
    'left_joint_2': -0.03,
    # ...
}

# 在读取位置时应用偏移
actual_position = encoder_position + joint_offset[joint_name]
```

## 9. 安全注意事项

### 9.1 设置零位前

```
⚠️ 确保人员撤离
- 机械臂可能轻微运动

⚠️ 手放在急停按钮上
- 发现异常立即按下

⚠️ 检查机械臂姿态
- 确保在安全位置
- 避免碰撞
```

### 9.2 设置零位中

```
⚠️ 不要触摸机械臂
- 电机已使能,有力矩输出

⚠️ 观察异常情况
- 异常声音
- 剧烈抖动
- 冒烟
```

### 9.3 设置零位后

```
✓ 回零测试
- 确认零位设置正确

✓ 低速测试
- 先低速运动测试
- 确认无异常后提高速度
```

## 10. 故障排查

### 10.1 电机ID冲突

**症状:**
- 某些电机行为异常
- 控制一个电机时另一个也动

**解决:**
```bash
# 1. 断开所有电机,逐个连接
# 2. 检查每个电机ID
# 3. 修改冲突的ID(需要电机配置工具)
```

### 10.2 零位无法保存

**症状:**
- 设置零位后,重启又恢复原值

**可能原因:**
- 电机Flash写入失败
- 电源不稳定

**解决:**
```bash
# 1. 确保电源稳定(48V±2V)
# 2. 重新设置零位
# 3. 设置后等待5秒再断电
# 4. 如果仍然失败,联系技术支持
```
