# 04_硬件故障排查

## 硬件问题诊断与解决指南

本文档提供OpenArmX系统常见硬件问题的诊断方法和解决方案。

## 1. 故障排查流程

### 1.1 系统化排查方法

```
1. 观察现象
   ↓
2. 定位子系统(电源/CAN/电机)
   ↓
3. 逐步排查
   ↓
4. 隔离问题
   ↓
5. 验证解决方案
```

### 1.2 安全第一原则

```
在排查过程中:
✓ 断电后进行物理检查
✓ 手放在急停按钮上
✓ 从小范围开始测试
✓ 记录所有操作
✓ 不确定时寻求帮助
```

## 2. 电源问题

### 2.1 完全无电

**症状:**
- 所有电机无响应
- 电源指示灯不亮
- 无任何声音

**排查步骤:**

```bash
# 1. 检查市电输入
# - 插座是否有电
# - 电源开关是否打开

# 2. 检查电源适配器
# - 指示灯是否亮
# - 使用万用表测量输出电压

# 3. 测量输出电压
万用表DC档,测量48V输出端
正常: 48V ± 2V
异常: 0V或远低于48V

# 4. 检查保险丝
- 查看是否熔断
- 更换相同规格保险丝

# 5. 检查线缆连接
- 接线端子是否松动
- 线缆是否断裂
```

**常见原因:**
- ❌ 市电未接通
- ❌ 电源开关未打开
- ❌ 保险丝熔断
- ❌ 电源故障

### 2.2 电压不稳定

**症状:**
- 电机间歇性掉线
- 电压波动大
- 电机运行时电压下降明显

**排查:**

```bash
# 使用万用表监控电压
# 空载: 48V ± 1V
# 负载: 应不低于46V

# 如果电压下降>2V:
# 1. 电源功率不足
# 2. 线缆压降过大(线太细或太长)
# 3. 电源老化
```

**解决方法:**
- 更换更大功率电源(≥1500W)
- 使用更粗的电源线(≥16AWG)
- 缩短电源线长度
- 检查接触电阻

### 2.3 急停无法解除

**症状:**
- 急停按钮解除后仍无电
- 需要多次操作

**检查:**

```bash
# 1. 检查急停按钮触点
万用表电阻档:
- 按下: 触点断开(∞)
- 解除: 触点闭合(<1Ω)

# 2. 检查继电器
- 是否卡死
- 触点是否氧化

# 3. 清洁触点
使用电气清洁剂喷洗
```

## 3. CAN通信问题

### 3.1 CAN接口无法启动

**症状:**
```bash
$ sudo ip link set can0 up
Error: Cannot find device "can0"
```

**排查:**

```bash
# 1. 检查USB连接
lsusb | grep -i can
# 或
lsusb | grep -i peak

# 应该看到CAN设备

# 2. 检查驱动加载
lsmod | grep can
lsmod | grep peak_usb

# 3. 查看dmesg
dmesg | tail -50 | grep -i can

# 4. 重新插拔USB
# 拔出等待5秒,重新插入

# 5. 尝试不同USB端口
```

**常见原因:**
- USB线缆故障
- USB端口供电不足
- 驱动未加载
- CAN转换器硬件故障

### 3.2 CAN通信间歇性中断

**症状:**
- 通信时好时坏
- RX/TX错误计数器增加
- 电机偶尔无响应

**诊断:**

```bash
# 1. 监控错误计数器
watch -n 1 'ip -s link show can0'

# 观察RX errors和TX errors
# 如果持续增加,说明通信质量差

# 2. 监听总线
candump can0 -e

# -e参数显示错误帧
# 如果出现ERROR FRAME,说明总线有问题

# 3. 检查总线利用率
# 使用can-utils的canbusload
canbusload can0@1000000 -p

# 如果利用率>80%,可能拥塞
```

**可能原因和解决:**

**1. 终端电阻问题**
```bash
# 测量CAN_H和CAN_L间电阻
# 应为60Ω(两个120Ω并联)

解决:
- 添加缺失的终端电阻
- 移除多余的终端电阻
```

**2. 线缆质量差**
```
解决:
- 更换屏蔽双绞线
- 确保线缆长度<5m
- 远离强电线缆(>30cm)
```

**3. 波特率过高**
```bash
# 降低波特率测试
sudo ip link set can0 down
sudo ip link set can0 type can bitrate 500000
sudo ip link set can0 up
```

**4. 接触不良**
```
解决:
- 检查所有连接器
- 紧固螺丝端子
- 清洁触点
```

### 3.3 Bus-Off状态

**症状:**
```bash
$ ip link show can0
can0: ... state BUS-OFF
```

**恢复:**

```bash
# 1. 重启接口
sudo ip link set can0 down
sudo ip link set can0 up

# 2. 手动重启
sudo ip link set can0 type can restart

# 3. 如果频繁Bus-Off:
# - 检查物理层(终端电阻、线缆)
# - 降低波特率
# - 检查电机是否短路
```

### 3.4 左右臂反向(PCAN特有)

**症状:**
- can0控制了右臂
- can1控制了左臂

**解决(PCAN设备):**

```bash
# 方法1: 重启PCAN模块
sudo rmmod peak_usb
sudo modprobe peak_usb

# 方法2: 拔插USB接口
# 拔出两个PCAN,等待5秒
# 先插入左臂(can0),再插入右臂(can1)

# 方法3: 在软件中交换can0和can1
# 修改motor_mapping.yaml
```

## 4. 电机问题

### 4.1 单个电机无响应

**症状:**
- 其他电机正常
- 某一个电机完全无响应

**排查:**

```bash
# 1. 使用check_motor_status.py
python3 check_motor_status.py

# 查看该电机是否列出

# 2. 监听CAN总线
candump can0 | grep "ID对应的帧"

# 如果看不到该ID的帧,说明电机未通信

# 3. 检查物理连接
# 断电后:
# - CAN线是否连接
# - 电源线是否连接
# - 连接器是否插紧

# 4. 单独连接测试
# - 断开其他电机
# - 仅连接该电机
# - 查看是否能通信

# 如果单独连接能通信:
# - 可能是ID冲突
# - 或总线负载问题

# 如果单独也不通信:
# - 电机CAN接口故障
# - 电机未供电
# - 电机硬件故障
```

### 4.2 电机抖动

**症状:**
- 电机持续抖动
- 发出噪声
- 无法稳定在目标位置

**原因和解决:**

**1. KP参数过大**
```python
# 降低KP值
# 原值: KP=50
# 调整: KP=20-30
```

**2. KD参数过小**
```python
# 增加KD值
# 原值: KD=0.5
# 调整: KD=1.0-1.5
```

**3. 机械卡顿**
```
检查:
- 关节是否有杂物
- 轴承是否损坏
- 齿轮是否磨损
```

**4. 编码器故障**
```bash
# 检查位置反馈是否跳变
python3 check_motor_status.py

# 观察位置值是否稳定
# 如果静止时位置大幅跳变,编码器故障
```

### 4.3 电机过热

**症状:**
- 温度>80°C
- 电机发烫
- 性能下降

**立即处理:**

```bash
# 1. 停止运动
python3 dis_all_motors.py

# 2. 让电机自然冷却
# 不要用水或风扇快速冷却

# 3. 等待温度<50°C
```

**原因分析:**

```
可能原因:
1. 负载过大
2. KP/KD参数过高(持续力矩输出)
3. 运动速度过快
4. 散热不良
5. 电机故障
```

**解决方法:**

```
□ 降低负载
□ 降低KP/KD参数
□ 降低运动速度和加速度
□ 增加散热(风扇)
□ 检查电机是否有短路
□ 如仍过热,更换电机
```

### 4.4 电机失控

**症状:**
- 电机高速旋转无法停止
- 不受控制运动
- 超出限位

**紧急处理:**

```
1. 立即按下急停按钮! 🔴
2. 断开48V电源
3. 等待电机完全停止
4. 不要靠近机械臂
```

**事后分析:**

```bash
# 1. 检查日志
ros2 log

# 2. 检查参数配置
# - KP是否异常大
# - 目标位置是否错误

# 3. 检查零位
python3 check_motor_status.py

# 4. 重新设置零位和参数后测试
```

### 4.5 电机无力

**症状:**
- 电机可以运动但力矩小
- 无法承载负载
- 容易被手推动

**检查:**

```python
# 1. 检查KP参数
# KP过小会导致无力
# 增加KP值: 10 → 20 → 30

# 2. 检查电机是否正常使能
python3 check_motor_status.py

# 3. 检查电源电压
# 电压过低会导致输出扭矩小
```

## 5. 机械问题

### 5.1 关节卡顿

**症状:**
- 某个关节运动不顺畅
- 有阻力感
- 运动过程中有停顿

**检查:**

```
1. 断电后手动转动关节
   - 应该顺滑
   - 阻力均匀

2. 检查机械连接
   - 螺丝是否松动
   - 零件是否错位

3. 检查轴承
   - 是否磨损
   - 是否缺润滑

4. 检查传动机构
   - 齿轮是否啮合良好
   - 有无杂物
```

### 5.2 机械松动

**症状:**
- 运动时有异响
- 关节晃动
- 定位精度下降

**解决:**

```bash
# 断电后:

1. 检查所有固定螺丝
   - 使用扭力扳手按规定扭矩紧固
   - 特别是电机固定螺丝

2. 检查连接件
   - 法兰连接
   - 轴套固定

3. 使用螺纹锁固胶(可选)
   - Loctite 243(中强度)
   - 防止螺丝松动
```

### 5.3 机械碰撞痕迹

**症状:**
- 外壳有划痕或凹陷
- 零件变形

**处理:**

```
1. 检查受损程度
   - 表面划伤: 可继续使用
   - 零件变形: 需要更换

2. 检查内部是否受损
   - 电机是否受冲击
   - 传动机构是否正常

3. 如有严重碰撞:
   - 联系技术支持
   - 可能需要专业检修
```

## 6. 传感器问题

### 6.1 编码器故障

**症状:**
- 位置读数跳变
- 零位丢失
- 运动轨迹异常

**诊断:**

```bash
# 监控位置反馈
python3 << EOF
import can
import time
from lib.motor_control import *

bus = can.interface.Bus(channel='can0', bustype='socketcan')

for i in range(100):
    # 读取电机1位置
    pos = read_motor_position(bus, 1)
    print(f"位置: {pos:.4f} rad")
    time.sleep(0.1)
EOF

# 观察位置是否稳定
# 静止时应该基本不变(±0.001 rad)
```

**如果位置跳变:**
- 编码器磁铁松动
- 编码器芯片故障
- 干扰问题

**解决:**
- 检查磁铁固定
- 更换编码器
- 改善屏蔽

### 6.2 温度传感器异常

**症状:**
- 温度读数异常(负值或>100°C)
- 温度不随负载变化

**检查:**

```bash
python3 check_motor_status.py

# 观察温度值
# 正常: 环境温度+10~50°C
# 异常: <0°C 或 >100°C
```

**如果异常:**
- 传感器故障(可继续使用,但无温度保护)
- 更换电机

## 7. 电气干扰

### 7.1 现象

- CAN通信不稳定
- 电机偶尔误动作
- 控制计算机死机/重启

### 7.2 减少干扰措施

**1. 接地**
```
□ 电源外壳接大地
□ 机器人基座接大地
□ 控制计算机接大地
□ 使用同一接地点
```

**2. 屏蔽**
```
□ 使用屏蔽CAN线缆
□ 屏蔽层单端接地(CAN转换器端)
□ 电源线使用滤波器
```

**3. 隔离**
```
□ 电源线和信号线分开走线(>30cm)
□ 避免平行走线
□ 交叉时垂直交叉
```

**4. 滤波**
```
□ 电源输入加EMI滤波器
□ 电机端加共模扼流圈(如有需要)
```
