## 2. CAN总线诊断

### 2.1 CAN接口管理

```bash
# 查看CAN接口
ip link show can0
ip link show can1

# 启动CAN
sudo ip link set can0 type can bitrate 1000000
sudo ip link set can0 up

# 关闭CAN
sudo ip link set can0 down

# CAN统计信息
ip -s link show can0
ip -s -d link show can0

# 清除错误计数
sudo ip link set can0 down
sudo ip link set can0 up
```

### 2.2 CAN通信测试

```bash
# 监听CAN消息
candump can0

# 带时间戳
candump can0 -t A

# 只看错误帧
candump can0 -e

# 过滤特定ID
candump can0,001:7FF

# 保存日志
candump -l can0

# 发送测试消息
cansend can0 001#0102030405060708

# 生成测试流量
cangen can0 -g 10 -I 1

# 查看CAN模块
lsmod | grep can

# 加载CAN模块
sudo modprobe can
sudo modprobe can_raw
```

### 2.3 USB-CAN设备

```bash
# 列出USB设备
lsusb

# USB设备详情
lsusb -v

# 内核消息
dmesg | grep -i can
dmesg | grep -i usb

# USB设备节点
ls -l /dev/tty* | grep USB
```

## 3. ROS2诊断

### 3.1 节点与话题

```bash
# 列出节点
ros2 node list

# 节点信息
ros2 node info /controller_manager

# 列出话题
ros2 topic list

# 话题信息
ros2 topic info /joint_states

# 话题频率
ros2 topic hz /joint_states

# 话题延迟
ros2 topic delay /joint_states

# 话题内容
ros2 topic echo /joint_states

# 话题带宽
ros2 topic bw /joint_states

# 发布测试消息
ros2 topic pub /test std_msgs/msg/String "data: 'test'"
```

### 3.2 服务与参数

```bash
# 列出服务
ros2 service list

# 服务类型
ros2 service type /controller_manager/list_controllers

# 调用服务
ros2 service call /controller_manager/list_controllers controller_manager_msgs/srv/ListControllers

# 列出参数
ros2 param list

# 获取参数
ros2 param get /controller_manager update_rate

# 设置参数
ros2 param set /controller_manager update_rate 200

# 导出参数
ros2 param dump /controller_manager > params.yaml
```

### 3.3 Action

```bash
# 列出Action
ros2 action list

# Action信息
ros2 action info /left_arm_controller/follow_joint_trajectory

# 发送Action Goal
ros2 action send_goal /left_arm_controller/follow_joint_trajectory control_msgs/action/FollowJointTrajectory "{trajectory: {joint_names: ['left_joint_1']}}"
```

### 5.2 Python快速诊断

```python
#!/usr/bin/env python3
import can
from robstride import Motor

# 连接CAN
bus = can.interface.Bus(channel='can0', bustype='socketcan')

# 测试电机
for motor_id in range(1, 9):
    try:
        motor = Motor(bus, motor_id)
        fb = motor.get_feedback()
        print(f"Motor {motor_id}: ✓ 位置={fb['position']:.3f}")
    except:
        print(f"Motor {motor_id}: ✗ 无响应")
```

## 10. 快速诊断脚本

### 10.1 完整系统检查

```bash
#!/bin/bash
# 完整系统诊断

echo "========== OpenArmX 系统诊断 =========="

echo -e "\n[1/7] 操作系统"
uname -r
lsb_release -d

echo -e "\n[2/7] CAN接口"
ip link show can0 | grep -E "state|UP"
ip link show can1 | grep -E "state|UP"

echo -e "\n[3/7] ROS2环境"
echo "ROS_DISTRO: $ROS_DISTRO"
echo "DDS: $RMW_IMPLEMENTATION"

echo -e "\n[4/7] ROS2节点"
ros2 node list | wc -l
echo "个节点运行中"

echo -e "\n[5/7] 控制器状态"
ros2 control list_controllers

echo -e "\n[6/7] 关节状态频率"
timeout 3 ros2 topic hz /joint_states 2>/dev/null || echo "无joint_states"

echo -e "\n[7/7] 系统负载"
uptime

echo -e "\n========== 诊断完成 =========="
```

### 10.2 故障速查

```bash
#!/bin/bash
# 快速故障检查

echo "快速故障检查..."

# CAN检查
echo -n "CAN接口: "
if ip link show can0 | grep -q "UP,LOWER_UP"; then
    echo "✓ can0正常"
else
    echo "✗ can0异常"
fi

# Controller Manager
echo -n "Controller Manager: "
if ros2 node list | grep -q controller_manager; then
    echo "✓ 运行中"
else
    echo "✗ 未运行"
fi

# 控制器
echo -n "左臂控制器: "
if ros2 control list_controllers | grep -q "left_arm_controller.*active"; then
    echo "✓ 激活"
else
    echo "✗ 未激活"
fi

# 关节状态
echo -n "关节状态: "
if timeout 2 ros2 topic echo /joint_states -n 1 &>/dev/null; then
    echo "✓ 有数据"
else
    echo "✗ 无数据"
fi

echo "检查完成"
```

## 12. 总结

### 12.1 常用命令组合

**系统启动:**
```bash
python3 en_all_can.py && python3 en_all_motors.py && ros2 launch openarmx_bringup openarm.launch.py
```

**快速检查:**
```bash
ip link show can0 && ros2 control list_controllers && ros2 topic hz /joint_states
```

**完全重启:**
```bash
python3 dis_all_motors.py && python3 dis_all_can.py && sleep 2 && python3 en_all_can.py && python3 en_all_motors.py
```

### 12.2 诊断清单

```
□ CAN接口UP (ip link show)
□ CAN无错误 (ip -s link show)
□ 电机响应 (check_motor_status.py)
□ Controller Manager运行 (ros2 node list)
□ 控制器active (ros2 control list_controllers)
□ joint_states发布 (ros2 topic hz)
□ CPU<80% (htop)
```
