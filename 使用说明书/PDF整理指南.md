## 🔧 转换工具推荐

### 方案一: Pandoc (推荐)

```bash
# 安装 Pandoc
sudo apt-get install pandoc texlive-xetex texlive-latex-extra

# 合并转换为 PDF (使用中文支持)
pandoc \
  00_封面.md \
  01_目录.md \
  00_安全声明/*.md \
  01_系统概述/*.md \
  02_环境准备/*.md \
  03_硬件连接与配置/*.md \
  04_电机测试与调试/*.md \
  05_MoveIt运动规划/*.md \
  06_轨迹录制与回放/*.md \
  07_控制器配置/*.md \
  08_高级应用/*.md \
  09_故障排查/*.md \
  10_开发者指南/*.md \
  11_附录/*.md \
  -o OpenArmX_使用说明书_v1.0.pdf \
  --pdf-engine=xelatex \
  -V CJKmainfont="Noto Sans CJK SC" \
  -V geometry:margin=2.5cm \
  --toc \
  --toc-depth=2 \
  --number-sections
```

### 方案二: Markdown PDF (VS Code 插件)

1. 安装 VS Code 插件 "Markdown PDF"
2. 按照文件顺序依次转换每个 Markdown 文件为 PDF
3. 使用 PDF 合并工具 (如 pdftk) 合并所有 PDF:

```bash
# 安装 pdftk
sudo apt-get install pdftk

# 合并 PDF 文件
pdftk \
  00_封面.pdf \
  01_目录.pdf \
  [其他章节PDF文件...] \
  cat output OpenArmX_使用说明书_v1.0.pdf
```

### 方案三: 在线工具

使用在线 Markdown 到 PDF 转换工具:
- https://www.markdowntopdf.com/
- https://pdf.online/markdown-to-pdf

## ⚠️ 注意事项

### 1. 页码对应
- 目录中的页码是**预估页码**
- 实际转换后请根据真实页码更新目录
- 建议使用 Pandoc 自动生成目录 (--toc)

### 2. 中文支持
- 确保 PDF 引擎支持中文字体
- Pandoc 推荐使用 `xelatex` 引擎
- 指定中文字体: `Noto Sans CJK SC` 或 `SimSun`

### 3. 图片和资源
- 确保所有 Markdown 中引用的图片路径正确
- 建议使用相对路径引用 assets 文件夹中的资源

### 4. 格式统一
- 检查各章节标题层级一致 (H1, H2, H3)
- 确保代码块、表格格式正确
- 统一使用同一种 Markdown 语法规范
