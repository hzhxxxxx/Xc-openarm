# 02_电机使能与初始化

## 电机使能、禁用和初始化操作

本文档说明如何使能和禁用电机,以及零位设置。

---

## 1. 电机使能概述

### 1.1 什么是使能

**使能(Enable)**是让电机进入可控状态:
- 电机通电并保持位置
- 可以接收和执行控制命令
- 输出力矩抵抗外力

**禁用(Disable)**是让电机进入自由状态:
- 电机断电,无力矩输出
- 可以手动推动
- 不响应控制命令

### 1.2 使能管理脚本

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

ls -l *motors.py
# en_all_motors.py   - 使能所有电机
# dis_all_motors.py  - 禁用所有电机
```

## 2. 使能电机

### 2.1 使用en_all_motors.py

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

# 前提: CAN接口已启动
python3 en_all_can.py

# 使能所有电机
python3 en_all_motors.py
```

**输出示例:**
```
使能电机...
can0 ID 1: 使能成功
can0 ID 2: 使能成功
...
can1 ID 8: 使能成功
所有电机已使能
```

### 2.2 使能时的现象

**正常现象:**
- ✅ 电机轻微抖动(瞬间)
- ✅ 保持当前位置
- ✅ 温度略有上升

**异常现象(需要急停):**
- ❌ 剧烈抖动或高速旋转
- ❌ 异常声音
- ❌ 冒烟

### 2.3 安全注意事项

```
使能前:
□ 确认CAN接口已启动
□ 机械臂处于安全姿态
□ 人员已撤离工作区域
□ 手放在急停按钮上

使能后:
□ 不要触碰机械臂
□ 观察电机状态
□ 检查温度和声音
```

## 4. 零位设置

### 4.1 使用set_motor_zeros_setMIT.py

**前提条件:**
```
□ CAN接口已启动
□ 电机已使能
□ 机械臂手动调整到零位姿态
□ 各关节偏差<30度
```

**执行步骤:**

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

# 1. 启动CAN
python3 en_all_can.py

# 2. 使能电机
python3 en_all_motors.py

# 3. 手动调整机械臂到零位(双臂垂直下垂)

# 4. 设置零位
python3 set_motor_zeros_setMIT.py
```

**输出示例:**
```
读取当前电机位置...
can0 ID 1: 位置 0.05 rad (偏差: 2.9°) ✓
can0 ID 2: 位置 -0.03 rad (偏差: 1.7°) ✓
...
所有电机位置在允许范围内

设置零位...
can0 ID 1: 零位设置成功
can0 ID 2: 零位设置成功
...
完成! 请重启电机使零位生效
```

### 4.2 零位设置失败

**错误示例:**
```
错误: can0 ID 5 位置偏差过大
当前位置: 0.85 rad (48.7°)
允许偏差: ±0.52 rad (±30°)
请手动调整后重试
```

**解决方法:**
```bash
# 1. 禁用电机
python3 dis_all_motors.py

# 2. 手动调整ID 5电机到零位附近

# 3. 重新使能
python3 en_all_motors.py

# 4. 重新设置零位
python3 set_motor_zeros_setMIT.py
```

### 4.3 验证零位

```bash
# 重启电机使零位生效
python3 dis_all_motors.py
sleep 2
python3 en_all_motors.py

# 回零测试
python3 control_motor_gohome.py

# 观察机械臂是否回到正确的零位姿态
```

## 5. 回零操作

### 5.1 使用control_motor_gohome.py

```bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

# 执行回零
python3 control_motor_gohome.py
```

**功能:**
- 将所有关节移动到0位置
- 验证零位设置是否正确
- 作为运动测试的起始姿态

**参数说明:**
```python
# 脚本中的关键参数
target_position = 0.0  # 目标位置(rad)
kp = 20.0              # 位置增益
kd = 1.0               # 速度增益
duration = 3.0         # 运动时间(秒)
```

### 5.2 回零安全检查

```
执行前:
□ 确保周围无障碍物
□ 机械臂当前位置接近零位(避免大幅度运动)
□ 手放在急停按钮上

执行中:
□ 观察运动轨迹
□ 监听异常声音
□ 准备随时急停

执行后:
□ 检查最终姿态是否正确
□ 检查关节角度(应接近0)
```

## 7. 批量脚本

### 7.1 创建一键启动脚本

```bash
cat > ~/openarmx_start.sh << 'EOF'
#!/bin/bash
cd ~/openarmx_robotstride_ws-cc/src/motor_tests_openarmx_com

echo "===== OpenArmX 启动序列 ====="

echo "[1/3] 启动CAN接口..."
python3 en_all_can.py
if [ $? -ne 0 ]; then
    echo "错误: CAN启动失败"
    exit 1
fi

echo "[2/3] 等待5秒,请打开48V电源..."
sleep 5

echo "[3/3] 使能电机..."
python3 en_all_motors.py
if [ $? -ne 0 ]; then
    echo "错误: 电机使能失败"
    exit 1
fi

echo "===== 启动完成 ====="
echo "运行 'python3 check_motor_status.py' 检查电机状态"
EOF

chmod +x ~/openarmx_start.sh
```

**使用:**
```bash
~/openarmx_start.sh
```

### 7.2 创建一键关闭脚本

```bash
cat > ~/openarmx_stop.sh << 'EOF'
```

## 8. 常见问题

### 8.1 使能失败

**症状:** 某些电机无法使能

**排查:**
```bash
# 1. 检查CAN通信
candump can0

# 2. 检查该电机是否响应
python3 check_motor_status.py

# 3. 检查电机供电

# 4. 尝试单独使能该电机
```

### 8.2 使能后立即掉线

**症状:** 使能成功,但几秒后失去响应

**可能原因:**
- 电源功率不足
- CAN总线质量差
- 电机过载保护

**解决:**
```bash
# 1. 检查电源电压(应稳定在48V±2V)

# 2. 检查CAN错误计数
ip -s link show can0

# 3. 降低初始KP/KD值
```

### 8.3 零位设置后仍然偏差

**症状:** 回零姿态不正确

**解决:**
```bash
# 1. 确认设置零位时姿态正确

# 2. 重启电机
python3 dis_all_motors.py
sleep 5
python3 en_all_motors.py

# 3. 重新测试回零
python3 control_motor_gohome.py

# 4. 如仍然偏差,重新设置零位
```
